12. 반응형 디자인과 미디어 쿼리 소개

이번 시간에는 현대적인 웹 어플리케이션을 만들 때 중요한 요소라고 할 수 있는 반응형 디자인이라고 하는 흐름을 알아보겠다 우리가 만들어왔던 예제를 보고 있다 이런 기능을 추가해보자 화면이 작아짐에 따라 선이 없어지고, 위치들이 달라졌다. 화면이 일정 크기가 되면 원래처럼 요소들이 된다

즉, 화면의 크기에 따라서, 웹페이지의 각 요소들이 반응해서 최적화된 모양으로 바뀌게 하는 것, 그것을 그것을 반응형 웹, 또는 반응형 디자인(responsive web)이라고 부른다 웹은 상당히 곤욕스러운 시절을 겪어왔다 웹은 거의 모든 정보 시스템, 운영체제와 상관없이, pc이건 스마트폰이건, 가상현실이건 모든 시스템에서 동작하는 정보 시스템이다 수많은 형태의 화면에서 동작해야 한다는 것이다 그렇다보니 웹을 만드는 사람들은 여러 가지 화면에 대응되는 웹페이지를 만들기 위해서 몸부림을 쳐야 했다 웹을 개발하는데 있어서 상당한 걸림돌로 작용했고, 꽤 긴 시간동안 웹의 단점이었다 사람은 방법을 찾는다

그래서 오랜 시간 웹을 위해서 헌신한 사람들이 다양한 환경에서 적응할 수 있는 기술들을 발전시킨 결과 오늘날 반응형 디자인이라는 최신 흐름까지도 나타나게 되었다 반응형 디자인은 무엇인가. 화면의 크기에 따라서 웹페이지의 각 요소들이 반응해서 동작하는 것이다 반응형 디자인을 우리가 순수한 웹을 통해서, css를 통해서 구현하는 핵심적인 개념인 미디어 쿼리라는 것에 대해서 살펴보겠다

미디어 쿼리

화면 크기에 따라서 어떤 html의 요소를 보였다가, 안보였다가 해볼 것이다 그것을 하기 위해서 아무런 의미도 없는 'div' 태그 하나에다 한 단어를 감쌌다 


<body>
    NAVIGATION ARTICLE
</body>
div 태그를 잘보이게 하기 위해서 style 속성을 바꿔보자 

<style>
div {
    border:10px solid green;
    font-size:50px;
}
</style>
div 태그를 화면의 크기에 따라서 화면에 보이게도 하고 안보이게도 하면? 화면의 크기에 따라 디자인을 다르게 했다고 볼 수도 있다 이때 필요한 것이 '미디어 쿼리'다 어렵기 때문에 단계적으로 접근하자

화면의 크기가 800px 보다 작으냐 크냐에 따라서 태그를 보이게 안보이게 하려면 화면의 크기가 어느 정도인지 알아야 한다 검사 도구를 열어보면 화면 창을 늘리고 줄일 때 오른쪽 상단에 화면의 크기가 뜬다 화면이 800px보다 크다면/작다면으로 코딩하려고 한다 그러기 위해서 알아야 할 것은 스크린의 width > 800px라면, div 태그의 display를 none으로 하고 싶다 


    <body>
        NAVIGATION ARTICLE
    </body>
div 태그를 잘보이게 하기 위해서 style 속성을 바꿔보자 

    <style>
    div {
        border:10px solid green;
        font-size:50px;
    }
    screen width > 800px
        div{
            display:none;
        }
    </style>
미디어쿼리가 아니라 이건 가짜코드이기에 이걸 미디어 쿼리로 바꿔야 한다 화면의 크기가 800보다 크다는 것은, 화면의 크기가 최소한 800px이다라는 것과 같다 min은 최솟값이라고 할 때 이렇게 적어보자 

    <style>
    div {
        border:10px solid green;
        font-size:50px;
    }
    @media(min-width:800px) {
        div{
            display:none;
        }
    }
    </style>
이번에는 반대로 해보자. screan width ⁢ 800px로 해보자 

        <style>
        div {
            border:10px solid green;
            font-size:50px;
        }
        @media(min-width:800px) {
            div{
                display:none;
            }
        }
        </style>


13. 미디어 쿼리 써먹기

보니 800px보다 작아지면 본문이 읽기 이상해지는 것 같다. 800px을 기준으로 해서 클 때와 작을 때의 화면을 달리해보자 기존의 디자인은 그대로 살려두고, 코드도 내벼려두고 조건을 새로 추가한다 screen width < 800px(=최대 px이 800이다) 


    <style>
    div {
        border:10px solid green;
        font-size:50px;
    }
    @media(min-width:800px) {
        div{
            display:none;
        }
    }
    </style>
id값이 grid라고 되어있는 부분이 grid가 아니게 하면 된다 

        #grid {
            border:5px solid pink;
            display:grid;
            grid-template-columns: 150px 1fr;
            }
이 코드에서, 이렇게 바꿔준다. 

        <style>
    div {
        border:10px solid green;
        font-size:50px;
    }
    @media(min-width:800px) {
        #grid {
        display:block;
        }
    }
    </style>
id값이 grid인 태그의 display가 grid였던 것이 block으로 바뀌게 된다 block level element가 되면, 화면 전체를 쓰니까 자연스럽게 목차 오른쪽 상자에 위치할 수 없게 되면서 밑으로 내려오게 된다

세로선은 왜 존재하는가? 검사를 열어서 보면, ol태그의 속성 때문에 존재한다 저걸 그대로 카피해와서 가져와서 이렇게 바꿔준다 


        <style>
    div {
        border:10px solid green;
        font-size:50px;
    }
    @media(min-width:800px) {
        #grid {
        display:block;
        }
        ol {
            border-right:none;
            }
    }
    </style>
가로줄도 없애고 싶다면 이렇게 추가하자 

        <style>
    div {
        border:10px solid green;
        font-size:50px;
    }
    @media(min-width:800px) {
        #grid {
        display:block;
        }
        ol {
            border-right:none;
            }
        h1 {
            border-bottom:none;
            }    
    }
    </style>
더 다양하게 활용할 수 있겠지만, 여기서는 이만큼만 다루고, 실제로 필요할 때 '아 미디어 쿼리를 공부하면 되겠구나'하고 느끼고 새롭게 공부하자


14. css 코드의 재사용

우리가 지금까지 만든 것을 다른 페이지들에게도 전파해야 한다 css 코드들을 다른 페이지에 전파해보자

vsCODE의 explorer탭(파일목록 탭)의 html 파일을 오른쪽 클릭해서 split right 클릭하고 나란히 보면서 고치자

극단적인 상황을 생각해보자 css 코드를 여러 페이지에 적용할 때, 조금 까다로웠다. 가지고 있는 웹페이지가 1억개라고 한다면, 이 웹페이지의 디자인을 바꿔야 한다? 모든 a 태그의 'text-decoration:none;'으로 바꿔달라. 라고 한다면? 이런 문제는 우리가 작성한 코드, css 코드가 모든 웹페이지에서 중복해서 등장하기 때문이다

코딩을 잘할 때 필요한 역량, '중복을 제거하는 것'이 필요한 때이다 중복된 파일을 제거해서 프로그램, 코딩이 얼마나 효율적인지를 다시 한번 확인해볼 것이고, 이걸 통해서 느껴야 할 것이 중복의 제거가 중요하고, 중복의 제거라는 것이 새로운 기술이 출현하는데에도 큰 역할을 했겠구나 하는 부분에 공감하자

모든 웹페이지에서 중복되고 있는 css 코드를 style 코드를 제외한 순수한 css 코드만을 복사해서 'style.css'파일을 만들고 붙여넣는다. 그리고 기존의 웹페이지에는 style태그 자체를 다 지워버린다 그리고 이렇게 웹브라우저에게 얘기한다. '웹브라우저야, 커서가 껌벅거리는 위치에는 style.css라고 하는 별도의 파일에 저장되어있는 css를 다운받아서, 원래 그 코드가 있었던 것처럼 동작해'라고 한다 이게 바로 'link'라는 태그이다 


            <meta charset="utf-8">
            <link rel="stylesheet" href="style.css">
            </head>
link 태그를 통해서 이 웹페이지가 어떤 css 파일과 연결되어있는지를 브라우저에게 알려줄 수 있다는 것이다 이렇게 표시하게 되면, 1.html이라는 웹페이지는 style.css 파일과 연결(link)되어있고, 웹브라우저는 style.css라는 파일을 다운로드받아서 그것을 웹페이지에 적용하게 될 것이다

나중에 무엇인가를 바꾸고 싶다면, 공통적으로 사용되고 있는 style.css 파일로 가서 지워주면, 모든 웹페이지에 이 효과가 적용되는 폭발적인 효과를 얻게 된다 이렇게 했을 때 얻을 수 있는 효과는 css에서의 선택자를 활용해서 얻을 수 있는 중복의 제거에서 누리는 효과를 누릴 수 있게 된다 재사용성이 높아지고, 사용하는 입장에서는 내부적인 코딩의 원리를 몰라도 사용할 수 있게 되니 사용성도 높아진다 1억개의 웹페이지여도 css를 하나만 바꾸면 동시에 바뀌는 폭발적 효과를 얻게 된다는 것이 혁명적인 사건이었다 코드의 양도 줄어서 이 웹페이지를 다운로드받을 때, 인터넷에 사용료를 덜 낼 수 있다는 것도 매우 중요한 경제적 효과이다

물론, 웹페이지를 다운로드 받을 때, css를 안 다운 받는 것은 아니다 검사를 열어 확인해보자. 개발자도구를 켜고, network를 클릭하면 현재 보고 있는 페이지를 리로드했을 때 내부적으로 어떤 파일들을 웹서버에서 어떻게 다운받는지를 보여주는 기능이다 1.html파일을 찾았고, 이걸 보면서 쭉 웹브라우저를 해석하면서 link 태그를 만나고 style.css 파일을 다운받아야 한다는 것을 알아서 몰래 style.css 파일을 다운받아서 웹페이지에 적용한 것이다

네트워크적으로는 어떤 게 더 적은 트래픽을 사용하는 것일까 웹페이지 내에 넣는 것이 더 효율적이긴 하다 그러나 지금부터 설명할 테크닉 때문에 그렇지 않다 캐싱 때문이다

캐싱은 저장하다라는 뜻인데 우리가 한번 style.css라는 파일을 받았다면, 저 파일을 바뀌기 전가지는 style.css라는 파일을 저 웹브라우저는 우리의 컴퓨터에 저장해놨다가, 그다음에 css 파일을 요청하면 저장된 파일을 가져와서 속도를 높일 수 있고,(왜냐하면 네트워크를 안쓰기 때문에) 사업자들은 돈을 덜 쓸 수 있는 굉장히 중요한 효과를 얻게 된다는 것이다 style.css 파일을 캐싱할 수 있게 되면 훨씬 더 빠르게 웹페이지를 보여줄 수 있으면서 네트워크의 트래픽(사용료)를 적게 낼 수 있다는 신기한 효과를 누릴 수 있다


15. 수업을 마치며

우리는 지금까지 css를 지배하는 중요한 규칙을 배웠다 그중에 중요한 것은 선택자와 속성이다 속성을 많이 알수록 더 풍부한 표현력을 얻게 되고, 선택자를 많이 알수록 내가 아는 속성을 더 정확하게 표현할 수 있게 된다 마치 사람의 언어에서 주어와 서술어의 관계는 선택자와 속성의 관계는 css를 지탱하는 두 개의 뿌리이다

이제 어떻게 해야할까 지금 이 시점은 공부를 그만하기에 좋은 타이밍이다 대신 공부한 것을 써먹기에 아주 좋은 타이밍일뿐만 아니라 실은 그래야 할 타이밍아다 공부는 많이 했는데 그것을 써먹지 않는다면 우리 뇌는 억울해한다 그러면 뇌가 수단과 방법을 가리지 않고 안한다 공부는 별로 안했는데 최대로 쓴다면 공부의 효과에 뇌가 감탄할 것이다 그러면 공부를 자꾸 하게 되어있다 공부를 안하면 뇌가 혹사당한다는 것을 알기 때문이다 예제를 이용해서 의미있는 웹사이트를 꾸며보자 웹사이트가 자신의 삶에서 중요하다면 기술적으로나 부족한 부분이 보일 것이다 그것을 채우기 위해서 고민하다보면 무관한 듯 떨어져 있던 개념들이 연결되기 시작하고, 그 연결이 충분히 촘촘해진다면 웹으로부터 알려주는 도움이 없는데도 내부로부터 스스로 깨달아가는 배움이 시작된다
