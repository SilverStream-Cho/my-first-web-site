'생활코딩' 학습기록: JS 1


1. 수업소개

HTML을 만들어낸 이후 인간의 욕심은 끝이 없어졌고, 그 수많은 욕심 중에 우리의 관심사는 사용자와 상호작용하는 웹페이지를 만들고 싶다는 욕망이다 HTML은 정적이다. 한번 화면에 출력되면 언제나 그 모습 그대로이다 그러나 우리가 알고 있는, 데스크탑이나 모바일에서 사용되는 여러 프로그램들(이를테면 게임 등)은 사용자의 조작에 반응해서 프로그램이 움직인다 사람들은 웹페이지를 게임처럼, 동적으로 사용자와 상호작용할 수 있게 되기 원했다 그래서 태어난 기술이 바로 '자바스크립트'이다

이제 웹은 HTML을 이용해서 웹페이지를 만든 후에, 그렇게 만들어진 웹페이지를 자바스크립트를 이용해서 사용자와 상호작용할 수 있도록 기능을 추가함으로써 HTML의 정보, 자바스크립트의 기능을 모두 갖춘, 전무후무한 유일무이한 시스템으로 성장했다 이를테면 우리가 알고 있는 수많은 웹사이트들이, 프로그램처럼 사용자와 상호작용하면서도 검색엔진을 통해서 검색된다는 것은 웹만이 가진 독창성이다 그 어떤 정보기술도 그렇지 않다

HTML이 멈춰있는 이미지를 그리는 것이라면, JS는 정보라고 하는 그 이미지를 마치 영화처럼 움직이게 한다 이것은 마법이다 이제 JS를 자유자재로 휘두르는 마법사가 되자

(노마드) html이 뼈대이고, css가 근육이자 어떻게 보이는지를 결정한다고 했다. 만약 웹사이트가 무언가를 하게끔 하려면, 뇌가 필요하다. 뼈대도 가지고 있고, 근육도 가질 수 있다. 그러나 뇌가 없다면, 아무것도 움직이지 않을 것이다. 이게 바로 js가 하는 일이다. 어떤 웹사이트는 똑똑해야 할 필요가 있다. 어떤 웹사이트들은 동적이어야 할 때가 있다. 무언가를 클릭했을 때 애니메이션이 생기거나, 스크롤 바, 등등 이런 것들을 동적 상호작용성(interactivity)라고 한다. 뭔가 클릭하면 어떤 일이 생긴다. 여길 클릭하면 어딘가로 이동하고, shuffle버튼을 누를 수도 있다. html과 css를 배운 후에, javascript, react, Vue.. 내가 배우고 싶은 뭐든 어떻게 사용해야할지 배울 것이다. (그것들은 웹사이트의 뇌가 될 것이다)


2. 수업의 목적

우리는 어떤 문법을 쭉 배워나가는 수업이 아니라, 하나의 기능을 구현하는 과정에서, 그걸 하기 위해서 필요한 문법을 챙기는 수업이다 실습이 중심이 되어서 문법을 익혀나가는 것이 부가적으로 따라간다 'night' 버튼을 누르면 페이지가 야간모드가 되고, 'day'를 누르면 주간모드가 되는 기능을 만들어볼 것이다 이것이 js가 할 수 있는 중요한 측면이다. 사용자와 상호작용한다는 것이다 즉 사용자가 night 버튼을 클릭했을 때 이것에 반응해서 웹페이지를 바꿔주고 있는 것이다

웹브라우저는 한번 화면에 출력이 되면, 자기 자신이 바꿀 수 있는 능력이 없다 그것을 해주는 것이 js이다 오른쪽 클릭을 해보면 '검사'라는 항목이 있다 검사를 통해 나온 것들 중에 탭에 'element'라는 게 있다. (태그라는 뜻이다 이 웹페이지를 구성하고 있는 html 태그들이 모이는 것을 알 수 있다)

거기서 night 버튼을 클릭했을 때, body 태그가 어떻게 바뀌는지를 보자 day 버튼을 클릭하면 이렇게 바뀐다 이제 만들어보자

'night'라고 적혀있는 버튼은 어떻게 만드냐. 'input'이라는 태그를 이용하면 저런 버튼을 만들 수 있다. 그런데 그 태그의 속성으로 'button'을 주어야 한다 그리고 value라고 적혀있는 'night'가 웹브라우저는 이 버튼의 이름으로 night로 쓰게 된다 그다음에 onclick이라는 속성이 나온다. 웹브라우저의 html 사용설명서에는 이렇게 적혀 있다 onclick이라는 속성의 값으로는, js가 와야한다. 라고 적혀있다 그리고 onclick이라고 하는 속성의 값으로 js를 넣어주면, 사용자가 onclick속성이 위치하고 있는 버튼을 클릭했을 때, js 코드를 그때 실행시킬 것입니다 라고 적혀있다 


        <input type>="button" value="night" onclick="
            document.queryselector('body').style.backgroundColor='black';
            document.queryselector('body').style.color='white';
            ">
이 문서(document)에서 body 태그('body')를 선택(selector)하는 코드가 이것이다 (문법을 절대신경쓰지 말고, 코드를 이해하려고 하지말고, 한국말을 이해하려고 하자) body 태그는 웹페이지 전체를 차지하는 아주 큼직한 태그이다(화면 전체라는 얘기이다) 즉, body 태그를 선택하는 코드가 첫줄이다 그리고 body 태그의 style 속성 값으로 backgroundcolor가 블랙으로 한다. 라고 한다 이 코드가 실행되면 화면 전체 색이 반전된다 그리고 개발자 도구에서는 이렇게 뜬다 

        <body style="background-color; black; color: white;">
day 버튼을 누르면 반대로 된다 

    <input type>="button" value="day" onclick="
        document.queryselector('body').style.backgroundColor='white';
        document.queryselector('body').style.color='black';
        ">
    

    <body style="background-color; white; color: black;">

교훈을 챙겨보자 1. js는 사용자와 상호작용하는 것을 하는 언어이다 2. 웹브라우저는 한번 화면에 출력되면 자기 자신을 바꿀 수 없다. (body 태그는 언제나 body 태그이다) 하지만 js를 이용해서 버튼을 클릭하면, js 코드에 따라서 body 태그에 스타일 속성이 추가되면서 body 태그의 디자인을 바꿔준다 이것이 js가 html을 제어하는 언어다. 라는 정의를 우리가 내리는 이유는 바로 이런 것 때문이다 이러한 특성이 웹페이지를 훨씬 더 동적으로, 다이나믹하게 만들어주는 특성을 웹페이지에 부여하는 것이라고 할 수 있다 이번 시간에 우리는 js 코드는 한 줄도 본 적이 없다고 치자 js로 뭘 할 수 있는지를 우리가 추론해볼 수 있도록, 이론적으로가 아니라 경험적으로 체험해볼 수 있도록 하는 수업이다


3. HTML과 JS의 만남: 기본

이번 시간에는 중요한 얘기를 할 것이다 기본적으로 js는 html 위에서 동작하는 언어이다(그렇게 시작했다 js는) 그래서 우리는 어떻게 하면 html이라는 컴퓨터 언어에 완전히 다른 문법을 가진 js를 낑겨넣을 것인가라는 얘기를 해보자 (아주 중요한 얘기이다)

새로운 파일을 열고, 기본 html 코드를 작성해보자 


    <!doctype html>
    <html>
    <head>
    <meta charset="utf-8">
        <title></title>
        <body>
            <script>
                document.write('hello world');
            <script>    
    </body>
    </html>
js를 넣는 방법은, 우선 1. 웹브라우저에게 html의 코드로 '지금부터 js가 시작됩니다'라고 알려줘야 한다 (그때 사용하는 방법이 'script'라는 태그이다) 이 태그는 script 안쪽에 js가 오도록 약속되어있기 때문에 웹브라우저는 이 안쪽에 있는 코드를 js로 해석한다 이를테면 hello world라는 글자를 출력하도록 js로 명령한다면 이렇게 된다 

    <!doctype html>
    <html>
    <head>
       <meta charset="utf-8">
        <title></title>
        <body>
            <h1>html</h1>
            hello world 
    </body>
    </html>
그러면 이것과 위의 차이는 무엇인가. 이런 정당한 생각을 하게 된다 

        <!doctype html>
        <html>
        <head>
           <meta charset="utf-8">
            <title></title>
            <body>
                <h1>html</h1>
                1+1
        </body>
        </html>
html에서 1+1은 영원히 1+1이 나올 것이다 하지만 js는 동적이다. 계산기처럼 동작할 수도 있는 다재다능한 녀석이다 숫자 2가 나온다 js는 숫자 1과 숫자 1을 더한 다음에 그 결과를 화면에 출력하는 능력을 갖고 있다 하지만 html은 정적이다 그런 것을 할 수 없다


4. HTML과 JS의 만남: 이벤트

이벤트라는 것을 이용할 것인데, 이것은 js가 사용자와 상호작용하는 데 핵심적인 역할을 한다(재미있는 주제니 집중하자)

기본적으로 html 코드를 만들었고, body 안에 버튼을 만들어볼 것이다 이때 사용하는 태그는 input 태그이다. 그리고 타입을 button으로 하면 버튼 모양이 된다 (버튼 내에 글씨를 쓰고 싶으면, value라는 속성을 사용하면 된다) 


    <!doctype html>
    <html>
    <head>
       <meta charset="utf-8">
        <title></title>
        <body>
            <input type="button" value="hi">
    </body>
    </html>
이제 하고 싶은 것은, 여기 있는 hi 버튼을 클릭했을 때 경고창이 뜨게 하고 싶다 경고창은 alert이라고 하는 기능이다(검색해보면, 어떻게 작동하는지 안다) 

        <!doctype html>
        <html>
        <head>
           <meta charset="utf-8">
            <title></title>
            <body>
                <input type="button" value="hi" onclick="alert('hi')">
        </body>
        </html>
이렇게 하고 실행시키면 경고창이 나온다.

어떻게 해서 이런 상황이 나온 것인지 다시 정리해보자 여기에 onclick이라는 속성을 썼는데, 이 속성은 아주 특별한 속성이다 html의 설명서에는, onclick의 속성값으로는 반드시 js가 와야한다고 써있다 그리고 onclick의 속성값은 웹브라우저가 기억하고 있다(onclick속성이 위치하는 태그, 즉 버튼을 사용자가 클릭했을 때, 기억하고 있던 js 코드를 js 문법에 따라 해석해서 그거에 따라 웹브라우저가 동작할 것입니다라는 뜻이다)

웹브라우저 입장에서는 alert('hi')를 기억하고 있다가 사용자가 이 버튼을 클릭하면 그때 실행해야지. 라고 생각하고 기다리고 있다는 것이다 실제 그런 일이 일어났을 때 실행을 한다

이렇게 웹브라우저 위에서 일어나는 일들을 뭐라고 하는가. 사건, 영어로는 event라고 한다 event가 일어났을 때 어떠한 js가 실행되도록 하는 것이 onclick이라고 한다

그러면, 웹브라우저에서 일어날 수 있는 이벤트에는 어떤 것들이 있을까. 만약에 모든 이벤트를 우리가 다 생각한다면 우리가 무한히 많은 이벤트가 있을거지만 그중에서도 기념할만한 이벤트들을 정해놨다 이를테면 무언가를 클릭하거나. 또는 이런 것이 있다

똑같은 input인데, type를 text로 바꾸면 텍스트 상자가 된다. 네모 안에 글씨를 입력할 수 있다 글씨를 입력하면 내용이 변한다 내용이 변했을 때, 라는 이벤트를 체크하는 이벤트도 있다 onchange속성을 활용해보자 


        <!doctype html>
        <html>
        <head>
           <meta charset="utf-8">
            <title></title>
            <body>
                <input type="text" value="hi" onchange="alert('changed')">
        </body>
        </html>
이렇게 하면 어떻게 되나 보자. abc하고 마우스포인터를 바깥쪽으로 뺄 때에 onchange 이벤트를 실행시키게 된다 쓸 데 없는 문구를 넣고, 다시 지운 다음에 빠져나오면 그때는 onchange 이벤트를 실행시키지 않는다

웹브라우저 위에서 일어나는 여러 가지 사건들 중에 기념할만한 일들을 몇 가지, 약 10개에서 20개 정도 이벤트들을 정해두고 있다 그 이벤트를 이용해서 사용자와 상호작용하는 웹사이트를 만들 수 있다라는 것이다

검색해보자 이를테면 여기에서 사용자가 어떤 키를 눌렀을 때 이벤트가 발생하게 하고 싶다. 그러면 이렇게 검색해볼 수 있을 것이다 'JavaScript' 'keydown event' 'attribute' 


            <!doctype html>
            <html>
            <head>
               <meta charset="utf-8">
                <title></title>
                <body>
                    <input type="text" onkeydown="alert('key down')">
            </body>
            </html>
텍스트 상자 내에서 키를 누르면 키를 눌렀다고 경고창이 뜬다

이번 시간에 배운 것은 on으로 시작하는 속성들이다. 이런 것들을 "event"라고 하고, 웹브라우저 위에서 일어나는, 사용자들에게 유용할만한 10개-20개 정도 정리해놓았다 이를 통해서 사용자와 상호작용할 수 있는 코드를 만들 수 있게 된다


5. HTML과 JS의 만남: 콘솔

js를 실행하는 또다른 방법을 살펴보자. 재미있고 실용적인 '콘솔'이라는 방법이 있다 지금까지 우리는 js를 실행하기 위해서 웹페이지, 파일을 만들었다 그러나 경우에 따라서는 파일이 아니라도, 어떤 코드를 실행해야 하는 아주 가벼운 상황들이 있다

홈페이지 내에서 오른쪽 마우스 클릭 - 검사(개발자 도구) - 메뉴 창 중에 concole창을 클릭한다 이 console을 이용하면 우리가 파일을 만들지 않고도 js 코드를 즉석에서 실행할 수 있다

활용법: 살다보면 계산이 필요할 때, 뭘 쓰나. 계산기를 쓴다 마찬가지로 살다보면 데이터를 처리해야 할 일들이 있다. 그런 경우에 습관적으로 console창을 열고, 여기에다 js를 실행시켜서 여러 가지 데이터들을 처리한다

웹페이지 내의 하나의 문단 텍스트를 콘솔 창에 붙여넣어보자. 이 텍스트가 몇 개의 글자로 이루어져 있는지를 알고 싶다고 하자. 세기가 어렵다 그런 경우에는 ''로 묶어준다 그러면 '' 사이에 있는 이 부분이 다 문자가 된다. 그리고 js에서는 이 문자의 숫자를 알려주고, 이 문자열의 문자 개수를 알려주는 명령어가 length이다('~~'.length)

'~~'.length 자체를 전체 alert로 감싸보자[alert('~~'.length)] 그러면 글자의 개수가 경고창에 뜬다. 이것도 우리가 js를 실행하는 방법이다

콘솔에서 실행시킨 js는 우리가 보고 있는 웹페이지 안에 삽입된 js인것처럼 동작한다 즉, 이 웹페이지를 대상으로 실행되게 된다 그 점에 착안해서 이런 것도 할 수 있다 댓글이벤트를 열었는데 1590명이 신청했고 나는 2명을 뽑아서 사은품을 줘야한다고 하자 그러면 수작업을 한다고 생각하면 어마어마하게 충분히 절망스러운 것이다 js가 구원할 수 있다 그러면 웹페이지 상에 신청한 댓글이 있는 그 페이지에 두고 오른쪽 버튼을 눌러서 개발자 도구를 연다 그리고 element를 보면, 이 웹페이지에 있는 html 코드들을 보여준다 이 상태에서 esc키를 누르면, 보는 것처럼 html 코드를 보는 것과 함께 console이 밑에 뜬다 다시 esc를 누르면 console이 없어진다

원하는 숫자만큼의 댓글을 랜덤하게 추출하는 코드를 전에 써두었다 이걸 카피해서 콘솔 빈 공간에 붙여넣기 하고 실행시키면, 코드에 의해서 가져온 데이터를 랜덤하게 셔플링하고 그중에 4명을 추출한다 (한번 실행했던 코드는 위쪽 화살표를 누르면 다시 실행할 수 있게 된다)

우리가 웹페이지를, js를 이용한다는 것은, 웹페이지를 만드는 것, 웹사이트를 만드는 것처럼 거대한 목표일 수도 있지만, 이미 만들어져 있는 웹사이트를, 나의 필요에 따라서 맥락적으로 간단하면서도 허접하고, 그렇지만 나한테는 딱 맞는 코드를 작성해서 그 문제를 해결할 수도 있다 우리가 코딩을 해서 얻을 수 있는 효용은 개발자가 될 수 있다 없다만이 아니라 나의 현실의 문제를 해결하는 수많은 단계의 심도들이 아날로그적으로 펼쳐져 있기 때문에 너무 큰 목표를 설정하는 것보다는 나의 현실의 문제를 가볍게 해결하는 방법을 잘 찾아보면 좋겠다
