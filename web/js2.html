6. 데이터타입 - 문자열과 숫자

컴퓨터 프로그래밍에서는 데이터를 처리하는 것이 아주 중요한 일이다. 그래서 컴퓨터에서는 데이터를 잘 처리하기 위해서 종류별로 데이터를 잘 분류하기 위해서 상당히 많은 노력을 했고, 하고 있다 그래서 js라는 컴퓨터 언어에는 어떤 형태의 데이터들이 있는가를 살펴보고, 그중에서 대표적이고 익숙할만한 데이터인 '문자'와 '숫자'에 대해서 알아보도록 하자

데이터 타입에 대해서 알아볼텐데, 한국어로는 '자료형'이라고 부른다 js에는 어떤 데이터 타입들이 존재하는지 우선 검색을 통해 살펴보자 'JavaScript data type'라고 검색하면 여러 결과가 나올 것이다 그중에서 JavaScript에는 6개의 데이터타입이 있고, 객체라는 것이 있다고 되어있다

  • 6개의 데이터 타입이 있다
    • Boolean
    • Null
    • Undefined
    • Number
    • String
    • Symbol(new in ECMAScript 6)
  • Object
여기있는 모든 것을 알 필요는 없다. Number가 숫자이고, String이라는 것이 문자열이라는 것을 알자 비교적 익숙한 데이터타입을 통해서 데이터타입이라는 것이 무엇인지를 좀 파악해보고, 각각의 데이터타입의 성격들을 살펴보자

한 웹사이트(어떤 페이지에서 console을 실행시켜도 상관없다)에 오른쪽 버튼을 눌러서 개발자도구로 들어가보자 console을 선택한다.

  • 숫자를 한번 표현해보겠다. 콘솔 창에 '12'라고 적는다(당연한 얘기겠지만, 사실 당연하지 않을 수 있다. 이건 나중에 얘기하겠다)
  • 'alert(1);'이라고 적어보자. 그러면 경고창에 숫자 1이 출력된다
  • 만약 'alert(1+1);'이라고 하면 결과는 2가 출력될 것이다
  • (경고창에 쓰는 것도 귀찮고, 경고창을 닫기도 귀찮다) 계산기처럼 편리하게 사용하려면 1+1이라고 적어보자. 그러면 결과는 숫자 2가 뜬다
숫자(number)라고 하는 데이터타입에서 아주 중요한 것은, '+' 등의 연산이다. +는 '왼쪽에 있는 값과 오른쪽의 값을 더해서 하나의 값을 만든다'라는 점에서 '이항연산자'라고 불린다(용어는 중요하지 않다) 이항연산자 중에서 산술을 하는 것이기에 '산술연산자'라고도 한다

또 다른 산술연산자에는 무엇이 있냐.

  • '-' : 빼기를 하는 산술연산자
  • '*' : 곱하기를 하는 산술연산자
  • '/' : 나누기를 하는 산술연산자
이렇게 우리가 숫자라고 하는 데이터타입이 어떻게 표현되는가와, 숫자와 숫자를 연산해서 어떤 결과를 만들어내는 산술 연산자 4개를 살펴봤다

다음에는 '문자'라는 데이터타입을 살펴볼 것이다 문자는 "로 시작해서 "로 끝나면 된다 만약 '를 시작하고 싶다. 그러면 '로 끝내면 된다

숫자 데이터 타입에서 중요한 것은 '산술연산자'이다. (이게 가장 많이 쓰인다) 문자열에서는 아주 중요한 기능이 된다. '여기있는 글씨가 몇 글자로 되어있는가'는 .length라고 하면 몇 글자인지 알려준다 


        > 'hello world'.length
        ⁢ 11
11글자라고 알려준다. 이렇게 한다.

그러나 문자는 엄청나게 많은 기능이 있어서 다 알려주기 힘들다 그래서 검색을 통해서 우리가 문자를 처리할 때 사용하는 명령들을 찾는 방법을 살펴본다 js에서 문자열은 영어로 string이다. (검색어 : JavaScript string) 이렇게 쳐서 mozila에서 운영하는 페이지에 들어가본다mozila

이 웹페이지의 왼쪽 목차를 보면, 우리가 살펴봤던 'string.length'가 있다 이런 것들을 properties라고 부른다(이걸 이해하는 것은 아직 과욕이니 듣고 흘리자) 여러 method라고 하는 형태도 존재한다. 재미있는거 위주로 해보자

'String.prototype.toUpperCase()'를 눌러보자

  • console.log("alphabet".toUpperCase()); // 'ALPHABET'
이런 식으로 쓰면 된다고 되어있는데, "문자열"을 쓰고, .toUpperCase()라고 쓰면, 결과가 'ALPHABET'이 된다는 뜻이다 실제로 확인해보면 소문자인 입력값이 결과값으로는 대문자로 출력된다는 것을 볼 수 있다

'String.prototype.indexOf()'를 눌러보자. 그러면 '찾고자 하는 값'을 넣으라고 되어있다

  • "hello world".indexOf('O'); // -1
  • "hello world".indexOf('o'); // 4
  • "hello world".indexOf('world'); // 6
'O'를 한번 넣어보자. 그리고 enter를 치면 '-1(찾을 수 없다는 뜻이다)'이 나온다 그런데 'o'를 넣으면, '4'가 나온다. 맨 앞부터 0, 1, 2, 3, 4번째에서 O가 나온다는 뜻이다 여기에 내가 'world'를 치면, 0, 1, 2, 3, 4, 5, 6번째에서 world라는 문자열이 시작된다는 뜻이다

공백을 없애주는 기능도 있다

  • ' hello '.trim()// "hello"
이런 것들을 이해하는 것보다 익숙해지도록 하자. 나중에 이론을 통해 경험적으로 배운 것을 compact하게 압축할 수 있는 기회가 있을 것이다. 그때까지는 이론을 모르는 것에 대해서 맘 편하게 가지자

  • 1 // 1
  • "1" // "1"
위는 숫자 데이터 타입이다. 아래는 문자열이다.
  • 1+1 // 2
  • "1"+"1" // "11"
문자열이냐, 숫자냐에 따라 그 결과는 많이 달라질 수 있기 때문에 항상 데이터타입을 정확하게 표현하는 것에 대해 관심을 가지고 잘 할 필요가 있다 문자열과 숫자를 서로 다른 데이터타입이다


7. 변수와 대입 연산자

개발자 도구-console을 켜서 실습을 진행해보자 변수라는 것은 '바뀔 수 있는 값'이다 예를 들어, 수학시간에 배웠던 것을 생각해보자

  • x=1; // 1
  • y=1; // 1
  • x+y; // ?
?에는 무엇이 들어갈까 2이다.

이것도 해보자.

  • x=1000 // 1000
  • x+1 // 1001
이 맥락에서 x라고 하는 것을 '변수', 영어로는 'variable'이라고 하고, '='를 대입연산자라고 한다 즉 오른쪽 항에 있는 값을 왼쪽에 있는 변수에 대입한다는 대입연산자이다
대입연산자는 또 하나의 특징이 있다. 좌항과 우항을 결합해서, 우항의 값을 만들어낸다. 라는 뜻이다 그러면, 이건 어떻게 될까
  • 1=2;
오류가 난다. 1에다가 2를 대입한 것이다. 당연히 1은 언제나 1이기 때문에 2를 대입하면 안되니 에러가 난다 x라고 하는 것은 대입연산자를 통해 값이 바뀔 수 있다라는 뜻을 가지고 있는 변수이다 하지만 여기있는 숫자 1은 언제나 '1'이기 때문에 저런 것을 뭐라고 하나. 바뀌지 않는다는 뜻에서 '항수', constant라고 한다

이제 변수를 왜 쓰는가에 대해서 생각해보자 이 이유는 너무 많기 때문에 한 가지를 콕 찝어서 말하기 어렵다 수학에서 만약 변수가 없다면, 공식이라고 하는 것을 인간은 사용하지 못할 것이다 그리고 프로그래밍에서 변수를 활용해서 할 수 있는 수많은 일들 중에, 그냥 예제로 변수의 힘을 느낄 수 있도록 하는 불완전한 예제를 줄 것이다

예를 들어 어떤 문자열을 쓴다고 해보자(내용은 아무런 의미가 없는 내용이다) 어떤 글을 쓴다. 그리고 그 글 안에는 '나의 닉네임'이 3군데 들어가 있다 그런데 극단적으로 사고해보자. 이 이름이 `1억개'라고 생각해보자. 그리고 굉장히 복잡하게 섞여있는 상태라고 생각해보자 그런데 이 글을 바꿔야 할 가능성이 생겼다. 이 글 안에 있는 이름들을, 예를 들어 leezche로 바뀌어야 한다 그러나 내가 다 바꿨는데 사람이라서 실수한다. 빠뜨렸다. 그러면 어떻게 하냐. 변수를 쓰는 것이다

name이라는 변수를 쓸 것이다

  • > name = '(닉네임)' // (실행안함)
여기서 enter 치면 실행되니 실행을 유보하자. (그럴 떈 shift + enter를 치면 된다) 그리고 위에 있던 본문을 복붙해놓는다.
  • > name = '(닉네임)'
  • > alert("~~~~~~ "+name+"~~~~~~)
닉네임이 있던 자리마다 "+name+"를 적는다. 그러면 변수처리가 된 것이다.

다음번에는 alert 부분은 전혀 신경쓰지 않아도 된다. 이제 name = '(바꾸고 싶은 부분)' 이 부분만 바꾸면 된다 이것이 바로 수많은 변수의 효용 중에 하나이다

변수를 사용할 땐, 가급적이면 변수 앞에 'var' 키워드를 쓰면 좋을 것이다(이게 조금 더 좋은 습관이다 variable의 약자이다)

  • > var name = '(닉네임)'
var의 장점과 단점을 알기 전까지는 항상 var을 쓰자


8. 웹브라우저 제어

지금부터는 js를 통해서 할 수 있는 일 중에 하나인, 웹브라우저를 제어하는 방법에 대한 구체적인, 중요한 테크닉들을 살펴볼 것이다

왼쪽에 보면 웹페이지를 뭘 하고 싶냐. night 버튼을 클릭했을 때 검은색으로 바꾸고 싶다 그러면 body라고 하는 태그가 웹페이지 전체를 감싸는 큰 태그이다. 저 태그에다 style이라는 속성을 쓰고, 


    <!doctype html>
    <html>
    <head>
    <meta charset="utf-8">
        <title></title>
        <body style="~~~">
            <script>
                document.write('hello world');
            <script>    
    </body>
    </html>
style이라는 속성의 속성값으로는 css라고 하는 html과 JavaScript와는 완전히 다른 컴퓨터 언어가 들어오도록 약속되어있다 이 언어의 역할은 디자인이다 

        <!doctype html>
        <html>
        <head>
        <meta charset="utf-8">
            <title></title>
            <body style="background-color:black; color:white;">
                <script>
                    document.write('hello world');
                <script>    
        </body>
        </html>
이렇게 하면 배경은 검은색, 글꼴은 흰색이 된다 night 버튼을 클릭했을 때 스타일 속성이 이렇게 될 수 있다면 우리가 하고자 하는 것을 할 수 있을 것이다 html은 이렇게 코드를 수정하면, 이렇게 할 수 있지만 언제나 이런 상태로 될 것이다 html은 한번 화면에 표시되면 자기를 바꿀 수 있는 능력이 없는 정적인 언어이다 하지만 js를 이용하면 버튼을 클릭했을 때 어떻게 되나 보자 (개발자 도구로 확인하니 버튼을 클릭하니 각 버튼마다 body 태그 내의 style 속성값이 변화하고 있는 것을 볼 수 있다) body 태그의 style 속성을 이렇게 주기 위해서는 우리가 이 속성을 주고자 하는 태그가 누구인가를 body 태그라는 것을 웹브라우저에게 알려주는 방법을 js로 어떻게 하는지 안다. 지금부터는 두 가지의 중요한 주제가 나온다
  1. css의 가장 본질적이고 중요한 문법들을 살펴볼 것이다
  2. js 를 이용해서 제어하고자 하는 태그를 선택하는 방법에 대해서 살펴볼 것이다


12. 제어할 태그 선택하기

js로 돌아와서, js, css, html을 잘 조합해서 우리의 최종적인 목적을 달성해보자 night 버튼을 누르면 색깔이 바뀌는 이유를 알아보자.

body라는 태그가 'night'버튼을 클릭했을 때, style 속성이 안에 생겨나면서 css가 지정하는 것을 통해서 디자인이 바뀔 수 있다 'day' 버튼을 클릭해도 또 다르게 바뀐다 


    <!doctype html>
    <html>
    <head>
    <meta charset="utf-8">
        <title></title>
        <body>
            <h1>⁢a href="index.html">WEB⁢/a>⁢/h1>
            <input type="button" value="day">
            <input type="button" value="night"> 
    </body>
    </html>
이렇게 하면 두 개의 버튼이 생겨난다. 버튼을 클릭했을 때 어떤 반응이 나타나게 하려면 '이벤트'가 필요하다(클릭에 대한 이벤트가 특히 필요할 것) 

        <!doctype html>
        <html>
        <head>
        <meta charset="utf-8">
            <title></title>
            <body>
                <h1>⁢a href="index.html">WEB⁢/a>⁢/h1>
                <input type="button" value="day" onclick="">
                <input type="button" value="night"> 
        </body>
        </html>
onclick="" 이 안에 js 코드를 쓰면 되는데, 길이가 길어지니까 enter를 해서 줄바꿈을 하고 넣는다

우리가 해야할 것은 굉장히 중요한 주제이다. 버튼이 클릭되었을 때 body 태그의 스타일 속성을 동적으로, 프로그래밍적으로, 상호작용에 의해서 넣으려고 한다 js의 문법에 따라서 웹브라우저에게 body 태그를 선택하도록 해야 한다 그것을 하기 위해서 검색을 해보자

내가 하고 싶은 것은 js에서 태그를 선택하고 싶은 것이다 (검색어: JavaScript select tag by css selector 찾고 싶은 것은 js에서 선택하고 싶은 것이다 select 뭐를? tag를 무엇으로? by id가 아니라 css의 선택자를 이용해서 - css에는 크게 태그 선택자/id 선택자/class 선택자가 있었다- 그중 css 선택자를 활용하면 훨씬 편하고 정교하게 선택할 수 있다)

  • [syntax] element = document.querySelector(selectors);
설명을 보니까 'document.querySelector'라는 명령을 이용해서 괄호 안쪽에다 selector를 넣으라고 되어있다
  • [example] In this example, the first element in the document with the class "myclass" is returned: > var el = document.querySelector(".myclass");
selector로는 .myclass라고 되어있고, .myclass라는 것의 의미는 잘 읽어보면 이 웹페이지에 있는 모든 태그 중에 class명이 myclass인 태그를 선택하는 선택자이다 그러면 실제로 입력해보자 

                <!doctype html>
                <html>
                <head>
                <meta charset="utf-8">
                    <title></title>
                    <body>
                        <h1>⁢a href="index.html">WEB⁢/a>⁢/h1>
                        <input type="button" value="day" onclick="
                            document.querySelector('')
                        ">
                        <input type="button" value="night"> 
                </body>
                </html>
이 문서에서(document), query(질의하다). 웹브라우저에게 질의한다. selector(여기서는 css의 selector를 뜻한다) 괄호 열고 우리가 원하는 것, body 태그를 선택하는 css selector는 body이다
  • 만일 내가 body에다 id 값을 주고, 'target'이라는 이름을 붙였다면 id가 target인 태그를 선택해야 하니까 ('#target')이라고 하면 된다

13. 프로그램, 프로그래밍, 프로그래머

지금까지 공부한 것을 통해서 js가 무엇인가. 또 js로 무엇을 할 수 있는가. 를 경험적으로 살펴보았다 이번 시간에는 js라는 프로그래밍 언어가 무엇인가를 이론적으로 되돌아보는 시간이다 그리고 프로그래밍 언어라는 것에 대한 좀 더 보편적인 의미를 음미해보는 시간이기도 하나이다

html과 js는 둘다 컴퓨터 언어이다 하지만, js는 컴퓨터 언어이면서 컴퓨터 프로그래밍 언어라고 하고, html는 컴퓨터 프로그래밍 언어라고 하지는 않는다 그 차이를 이해하기 위해서는 'program'이라는 말의 의미를 알면 좋겠다 어원을 찾아보면, 음악회 같은 곳에서는 시간이 흐름에 따라 음악이 연주되는 순서가 있다 이런 순서를 예전부터 program이라고 했다고 한다 program이라는 말의 중심에는 즉, 순서라는 의미가 깊숙이 자리잡고 있다 이 순서를 만드는 행위를 '프로그래밍'이라고 하고, 그 순서를 만드는 사람을 '프로그래머'라고 하는 것이다 'program', 'programming', 'programmer'라는 말은 소프트웨어 분야 뿐만 아니라 아주 많은 분야에서 사용되는 말이다 소프트웨어가 그것을 차용한 것이다

컴퓨터로 돌아와서 생각해보자. 컴퓨터에는 여러 가지 기능들이 있다 우리가 컴퓨터로 어떤 일을 하려고 할때, 그 기능들 중에 하나만 필요한 경우는 사실 없다 어떤 의도에 따라서 순서대로 여러 기능들을 작동시키는 것이 우리가 컴퓨터를 사용하는 일반적인 방법이다 이 순서대로 컴퓨터를 조작하는 것도 좋지만, 그 순서가 반복되고, 반복되는 과정에서 시간이 많이 걸리고, 또 자주 반복하고, 우리가 실수를 자꾸 한다면, 우리 마음속에서는 그 반복적인 작업을 하는 자신에 대해서 불쌍히 여기는 마음이 자라날 것이다 그 불행에 충분히 절망한 사람들이 이 불행을 어떻게 극복할 것이냐라는 고민을 하면서 컴퓨터 프로그래밍 언어라는 것을 만들었다

시간의 순서에 따라 실행되어야 할 기능을 프로그래밍 언어의 문법에 맞게 글로 적어두는 방식을 고안한 것이다 그리고 그 작업이 필요할 때 적어두었던 글을 컴퓨터에게 내밀면서, 이것을 실행해줘. 라고 하면 컴퓨터는 그 일을 할 것이다 바로 이것이 프로그램의 가장 본질적인 의미로 생각된다

html은 '웹페이지를 묘사하는 목적'의 언어이기 때문에 시간의 순서에 따라서 무엇을 할 필요가 없다 그래서 html은 시간의 순서에 따라 실행되도록 하는 기능을 가지고 있지 않다 하지만 js는 다르다. '사용자와 상호작용하기 위해서' 고안된 컴퓨터 언어이고, 시간의 순서에 따라 웹브라우저의 여러 기능이 실행되어야 하기 때문에 프로그래밍이라고 하는 형태를 띄고 있고, 바로 이런 점이 html과 js를 다르게 하는 중요한 특징이라고 생각한다

지금까지 우리는 여러 개의 명령어를 시간의 순서에 따라 배치해서, 컴퓨터의 기능이 실행되도록 하는 방법을 알아봤다 이 코드에서 보는 것처럼 body 태그의 배경색을 검은색으로 한 다음에, body 태그의 텍스트 컬러를 흰색으로 해. 라고 js라는 언어를 통해서 웹브라우저에게 명령하고 있는 것이다 이미 프로그래밍이라는 것이 무엇인가를 알고 있는 것이나 다름없다 사실 이것만으로도 놀랍도록 많은 것을 할 수 있다

그러나 사람의 욕심은 끝이 없다 처음에는 단순히 시간을 나열하는 것으로도 만족스러웠지만, 조건에 따라서 다른 순서의 기능이 실현되게 하고도 싶어지고, 또 반복해서 어떤 기능이 실행되도록 하고도 싶어졌다 또 이 코드가 복잡해짐에 따라, 순서의 배치가 복잡해짐에 따라 단순하게 잘 정리정돈하는 방법도 차차로 고안되기 시작했다 지금부터 우리는 이러한 인류의 여러 가지 욕심들, 그 욕심을 이루기 위해서 성취한 위대한 업적들, 흔적들을 따라가면서 컴퓨터 프로그래밍 언어라면, 어떤 언어이건 가리지 않고 가지고 있는 핵심적인 기능들, 조건문, 반복문, 함수. 이러한 기능들을 지금부터 살펴볼 것이다
