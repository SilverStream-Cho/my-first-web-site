14. 조건문 예고

지금부터 '조건문'이라고 하는 굉장히 중요한 주제에 대해서 살펴볼 것이다 조건문이라는 것은 하나의 프로그램이 하나의 흐름으로 가는 것이 아니라, 조건에 따라 다른 순서의 기능들이 실현되도록 하는 것이라고 할 수 있다 프로그래밍이라는 것이 제일 중요하고, 프로그래밍을 통해 반복적인 일을 하지 않도록 되었다면, 조건문이라는 것은 단순한 반복문이 아니라 아주 복잡한 업무까지도 해방되도록 하는 아주 혁명적인 도구이다 지금부터 살펴보자. 예제를 보자

'night'와 'day'라는 정말 대단한 기능을 구현했더니, 가만히 이걸 쓰다보니 불만족이 생기기 시작한다 어떤 불만족이 있을 수 있을까? 버튼이 두 개가 있는게 너무 불편하다 상태가 있는지에 따라 night와 day를 선택하는 것이 안 세련되어보인다 그래서 껌껌한 상태에서 버튼을 클릭하면 밝은 상태가 되고, 밝은 상태에서 버튼을 클릭하면 껌껌한 상태가 되는, 그런 기능을 구현하고 싶어졌다 이런 것을 '토글'이라고 부른다

그러면, 토글이라는 것을 구현하기 전에, 마법을 써서 토글 기능을 가진 버튼을 만들어보겠다 day를 누르면 밝아지고, 버튼이 night로 바뀐다. night를 누르면 다시 day로 바뀌면서 야간모드로 바뀐다 여기에서 가장 중요한 키워드는 'if'라는 조건문이다 이 조건문에 따라 만약 현재 day 모드라고 한다면, 아래의 코드가 실현되고, 현재 주간모드가 아니면 더 아래의 코드가 실행되는 것을 볼 수 있다 그리고 추가적으로 '비교연산자'라는 것도 살펴볼 것이다 이 비교연산자를 통해서 만들어지는 결과인 'boolean'이라는 것도 다음 시간에 살펴볼 것이다

만약 프로그래밍을 경험한 적이 있고, 조건문, 또는 비교연산자, boolean을 알고 있다면 이후의 js 문법에 대한 수업은 건너뛰고, 이 기능을 이용해서 최종적으로 토글 기능을 구현하는 쪽으로 건너뛰자 코딩이 처음이라면 계속 따라오면 된다


15. 비교연산자와 블리언

지금부터 js의 여러 기능 중에서 비교연산자라는 것과, 비교연산자를 통해 만들어지는 boolean이라는 새로운 데이터타입과 조건문을 연달아 같이 살펴볼 것이다 이 세 가지는 아주 깊은 관련성이 있기에 떨어뜨려놓고 얘기할 수 없다

실습을 진행하기 위해 기본적인 html 파일을 만들었다 


        <!doctype html>
        <html>
        <head>
        <meta charset="utf-8">
            <title></title>
            <body>
            <h1>Comparion operator & Boolean</h1>
        </body>
        </html>
h1 태그에 오늘 수업의 주제를 쓰고, 소개할 3개의 연산자를 하나씩 살펴보자 

        <!doctype html>
        <html>
        <head>
        <meta charset="utf-8">
            <title></title>
            <body>
            <h1>Comparion operator & Boolean</h1>
            <h2>===</h2>
                <script>
                    document.write();
                <script>    
        </body>
        </html>
'===' equal이 3개인 연산자는 무엇일까? 이것을 실행한 결과는 script로 표현해보자 

        <!doctype html>
        <html>
        <head>
        <meta charset="utf-8">
            <title></title>
            <body>
            <h1>Comparion operator & Boolean</h1>
            <h2>===</h2>
            <h3>1===1</h3>
                <script>
                    document.write(1===1);
                <script>    
        </body>
        </html>
이것은 웹페이지에다 저 괄호 안에 있는 정보를 출력하라는 것이다. 1===1은 무슨 뜻인가? 이는 '동등비교연산자'인데, 연산자의 왼쪽에 있는 값과, 오른쪽에 있는 값이 같느냐라는 뜻이다 같은가 다른가? 같다. 참이다. 그러면 'true'가 출력된다(h3로 무엇을 비교했는지도 적어주자) 

            <h3>1===2</h3>
                <script>
                    document.write(1===2);
                <script>
이 결과는 false가 나온다

우리가 음미해봐야할 부분은, '1===1' 이 부분에 대해서다. 이게 무엇이냐. 비교연산자이다. 그리고 이항연산자이다(좌항과 우항이 있고, 좌항과 우항을 결합해서 어떠한 데이터를 만든다) 1+1은 결과가 '2'이다. +라고 하는 이항연산자는 왼쪽과 오른쪽의 값을 더해서 전체가 2가 된다 === 연산자는 왼쪽의 값과 오른쪽의 값을 비교해서 '만약 같다면' 이것 자체가 뭐가 되냐? 'true'라는 값이 되는 것이다 그리고 '1===2'의 경우 둘이 다른데 같다고 하고 있다. 그러면 거짓이니까 'false'가 된다 즉 '==='라고 하는 비교연산자는, 좌항과 우항의 관계에 따라서 true, 아니면 false의 값을 만들어내는 연산자이다 그리고 true와 false의 두 가지 값을 묶어서 boolean이라고 부른다 지금까지 우리가 살펴봤던 데이터타입은 무엇이 있었나. number가 있었다 두번째는 문자열, string이 있었다 세번째는 boolean이 있었다. number라는 데이터 타입에는 몇 가지의 구체적인 데이터가 있을까 무한히 많다 물론 컴퓨터라는 기계의 한계에 의해서 실제로 무한히 많지 않지만, 일단 무한히 많다고 해보자.

string은 이 안에 몇 개의 문자가 이 안에 들어있을까. 엄청 많다. 인간의 언어만큼 boolean은 단 두 개의 데이터로 이루어져 있는 데이터 타입이다

boolean으로 인해서 인류는, 단순 반복에서 해방될 수 있게 되었다 boolean이라는 저것은, 영국의 수학자인 조지 부울이라고 하는 분의 수학적인 업적을 기리기 위한 것이었는데, 조지 부울 선생님도 후대에서 이 boolean이라는 수학에서의 논리학이 이렇게 사용될 것이라는 것을 잘 몰랐을 것이다(신기한 일이다) boolean을 만들어내는 연산자를 우리가 본 것이다(이게 어떤 실용적인 의미가 있느냐에 가려면 약간 시간이 필요하다)

이걸 이제 그대로 카피해서 이번에는, 이렇게 해보자 


        <h3>1<2</h3>
            <script>
                document.write(1<2);
            <script>
less than이라는 뜻에서 lt;이 비교의 뜻 기호 그대로이다 이 비교연산자는 좌항과 우항을 비교해서 두 개가 2가 더 크니까 참이 되니 이 부분 전체가 true가 된다

이 얘기 자체로는 쓸 수 있는게 아무것도 없다. 그러나 조건문과 반복문을 통해서 이 boolean이라는 것이 얼마나 혁명적인 가능성을 가지고 있는 관념인가에 대해서 감동받을 것이다


16. 조건문

지금부터 boolean과 비교연산자를 어디에 쓰는지 살펴보자 새로운 예제를 만들기 위해서 새로운 코드를 만들자 


        <!doctype html>
        <html>
        <head>
        <meta charset="utf-8">
            <title></title>
            <body>
            <h1>Conditional statementsn</h1>
        </body>
        </html>
이제 프로그램을 하나 짜볼 것이다. 아주 쓸모없지만, 일부러 아주 단순하게 만든 것이다 

        <h3>Program</h3>
            <script>
                document.write("1<br>");
                document.write("2<br>");
                document.write("3<br>");
                document.write("4<br>");
            <script>
br을 쓰면, 줄바꿈이 된다. 그렇게 해서 실행하면 1, 2, 3, 4가 한줄에 한 문자씩 출력된다(순서에 따라 실행된 것이다)

그다음에 'if'문을 한번 써볼 것이다. 아까 것을 그대로 카피해서 복붙한 다음에, '만약(if)'와 '그렇지 않으면(else)'를 넣었다 


            <h3>IF-true</h3>
                <script>
                    document.write("1<br>");
                    if(){
                        document.write("2<br>");
                    } else {
                        document.write("3<br>");
                    }
                        document.write("4<br>");
                <script>
if 뒤의 괄호에 (true)를 넣으면 결과가 어떻게 되나 보자 

        <h3>IF-true</h3>
            <script>
                document.write("1<br>");
                if(true){
                    document.write("2<br>");
                } else {
                    document.write("3<br>");
                }
                    document.write("4<br>");
            <script>
그러면 순서대로 실행되는데, 1이 실행되고, 2가 실행된 다음에, 3은 안나오고 4가 나왔다 즉, 실행의 순서가 바뀌었다(3과 관련된 코드를 무시하는 방향으로)

그러면 그대로 카피해서 이번에는 if 뒤의 괄호값을 (false)로 하고 실행해보자 


        <h3>IF-true</h3>
            <script>
                document.write("1<br>");
                if(false){
                    document.write("2<br>");
                } else {
                    document.write("3<br>");
                }
                    document.write("4<br>");
            <script>
이번에는 순서대로 1, 3, 4가 나왔다. 이 얘기는 1이 실행된 다음에, false니까 else 안에 있는 코드가 실행되고, 그다음에 4가 실행되었음을 알 수 있다

바뀐다

즉, if 문의 뒤에 따라오는 괄호 안에는, boolean 데이터 타입이 온다. (true or false) 그리고 true면, 첫번째 중괄호에 있는 코드들이 실행되고 두번째 중괄호는 무시된다 만약 false라고 하면, 첫번째 중괄호에 있는 코드들이 무시되고, else 안에 있는 코드가 실행된다 이것이 조건문을 이해하는 가장 중요한 요소이다(즉, boolean의 값이 무엇이냐에 따라서 실행되는 코드가 바뀐다)

물론 우리가 지금 짠 코드는, if뒤의 괄호에 true와 false를 직접 써서, 이 부분의 boolean 값을 직접 코딩되게 했다 그렇기에 앞은 언제나 true, 뒤는 언제나 false이기 때문에 완전히 사실은 쓸모없는 코드이다 왜냐하면 조건에 따라서 실행되는 코드가 달라지지 않기 때문이다

그러면 이제 우리가 해야할 것은, if 뒤에 오는 괄호 안에, boolean값이 true나 false로 들어가는데, 조건에 따라서 true가 오고, 조건에 따라서 false가 오도록 하면 된다

이번 시간에는 조건문의 형식에 대해서 알아보았고, 다음 시간에는 우리가 갖고 있는 예제에 조건문을 도입함으로서 얼마나 똑똑한 프로그램으로 성장할 수 있는지를 살펴보도록 하겠다


17. 조건문의 활용

지금까지 우리가 비교연산자, boolean, 조건문이라고 하는, 보기에 따라 조금 관념적일 수 있는 도구들을 살펴보았다 이 관념적인 도구들이 얼마나 현실에서 구체적인 도움을 우리에게 주는지 감상해보자

예제에서 night, day 버튼을 하나로 퉁칠 것이다(우선 버튼부터 만들어보자) 


    <!doctype html>
    <html>
    <head>
    <meta charset="utf-8">
        <title></title>
        <body>
            <h1>⁢a href="index.html">WEB⁢/a>⁢/h1>
            <input type="button" value="day">
            <input type="button" value="night"> 
    </body>
    </html>
  • <input type="button" value="day">
value값은 기본적으로 이 웹페이지가 열렸을 때는 배경이 흰색이니까, 클릭했을 때 night로 바뀐다는 의미에서 night로 바뀐다 night 버튼을 클릭했을 때, js를 이용해서 이 웹페이지 안에 있는 night 버튼의 value값이 현재 night 상태이다 만약에 이 버튼의 value값이 night라면, night버튼을 눌렀을 때 이 코드들이 실행이 된다고 해보자 

            <body>
                <h1>⁢a href="index.html">WEB⁢/a>⁢/h1>
                <input type="button" value="night" onclick="
                if night
                    document.querySelector('body').style.backgroundColor = 'black';
                    document.querySelector('body').style.color = 'white';
                else 
                document.querySelector('body').style.backgroundColor = 'white';
                document.querySelector('body').style.color = 'black';
            ">
            </body>
        </html>
만약 낮이라면, 이라고 적고, night가 아니라면, 으로 나눈다. (night가 아니라면, day버튼을 눌렀을 때의 코드 짜놓은 것을 넣어두면 된다)

이제 형식에 맞게 조건문을 써보자 


            <body>
                <h1>⁢a href="index.html">WEB⁢/a>⁢/h1>
                <input type="button" value="night" onclick="
                    if(night){
                        document.querySelector('body').style.backgroundColor = 'black';
                        document.querySelector('body').style.color = 'white';
                    } else {
                    document.querySelector('body').style.backgroundColor = 'white';
                    document.querySelector('body').style.color = 'black';
                    }
                 ">
            </body>
        </html>
()괄호 안이 주인공이 된다. 상황에 따라서 저 부분이 true이거나 false가 되게 하면 된다 그걸 하기 위해서 첫번째 해야할 것은, 현재 이 버튼의 value값이 무엇인지 알아낼 수 있어야 한다 그것을 하기 위해서 console을 이용한다

그다음에 버튼 태그의 id 값을 'night_day'로 할 것이다

  • <input id="night_day" type="button" value="night" onclick="
이렇게 해두고 console 창으로 가자

id값이 'night_day'인 저 element의 value값을 어떻게 알아내는지 검색을 통해 살펴보자 (검색어: JavaScript element get value)

  • var value = document.getElementById(id).value;
뒤는 잘 모르겠고, .value라는 property를 이용하면 된다고 되어있다

콘솔 창에서 이렇게 적어보자

  • > document.querySelector('#night_day')
이렇게 했을 때, 잘 데이터를 가져오는지 보자. 잘 데이터를 가져온다. 위쪽 화살표를 누르면 우리가 아까 눌렀던 코드로 재진입된다.
  • > document.querySelector('#night_day').value
이렇게 하면
  • > "night"
night를 가져온다 이제 이 태그를 활용하면 night_day의 value 값을 가져올 수 있다는 것을 알았다

이제 그 점에 착안해서 뭘 할까. 


            <body>
                <h1>⁢a href="index.html">WEB⁢/a>⁢/h1>
                <input type="button" value="night" onclick="
                    if(document.querySelector('#night_day').value === 'night'){
                        document.querySelector('body').style.backgroundColor = 'black';
                        document.querySelector('body').style.color = 'white';
                    } else {
                    document.querySelector('body').style.backgroundColor = 'white';
                    document.querySelector('body').style.color = 'black';
                    }
                 ">
            </body>
  • > if(document.querySelector('#night_day').value === 'night')
이렇다면 아래의 코드를 실행하고, 그렇지 않다면 주간모드에 해당되는 코드가 실행되라고 할 수 있겠다

그러나 night 모드로 바뀐 이후 더 변하지 않는다. 그래서 다음과 같이 수정한다 


            <body>
                <h1>⁢a href="index.html">WEB⁢/a>⁢/h1>
                <input id="night_day" type="button" value="night" onclick="
                    if(document.querySelector('#night_day').value === 'night'){
                        document.querySelector('body').style.backgroundColor = 'black';
                        document.querySelector('body').style.color = 'white';
                        document.querySelector('#night_day').value = 'day';
                    } else {
                    document.querySelector('body').style.backgroundColor = 'white';
                    document.querySelector('body').style.color = 'black';
                    document.querySelector('#night_day').value = 'night';
                    }
                 ">
            </body>


18. 리팩토링 - 중복의 제거

이번 시간엔 리팩토링에 대해서 알아볼 것이다 리팩토링은 factory는 공장이다 공장으로 다시 보내서 개선한다의 느낌 정도로 생각하자 refactoring은, 우리가 코딩을 하고 나면, 코드가 좀 비효율적인 면들이 생기기 마련이다 그러면 동작하는 것은 그대로 두고, 코드 자체를 아주 효율적으로 만들어서, 그 코드의 가독성을 높이고 유지보수를 편리하게 만들고, 중복된 코드를 낮추고, 이러한 방향으로 코드를 다시 개선하는 작업을 리팩토링이라고 한다 소프트웨어가 커지고 복잡해지는 데 있어서 여러분이 틈틈이 리팩토링을 해야지만 좋은 프로그램을 만들 수 있다 그래서 이번 시간엔 지금 우리가 만든 코드에서 비효율적인 것들을 제거하는 리팩토링의 사례를 보여보겠다

이 코드를 효율적으로 만들어보자 night 기능을 스크롤한 밑에서도 보내게 할 수 있도록 하고 싶다. 그러면 복붙해서 아래쪽에도 코드를 붙여넣어보자 그러나 동작은 하는데, 더 안바뀐다. 위쪽이 day로 value값이 바뀔 뿐이다(아래쪽 버튼은 night를 눌러도 night 그대로이다) 그러면 night_day라고 위가 되어있다면, 아래쪽은 night_day2라든가. 구분해줘야 한다 그러면 이걸 클릭할 때마다 정상적으로 동작한다

그러나 불편한 부분이 있다. 만약에 이런 버튼을 1억개를 복사한다면 1억개를 다 바꿔야 한다 그래서 onclick과 같은 이벤트 안에서 실행되는 코드들은 강조표시 안에서는 이 코드가 속해있는 태그, 즉 여기에서는 index 태그이다 그 태그를 가리키도록 약속되어있는 특수한 키워드가 있다 그게 바로 this이다 


        <body>
            <h1>⁢a href="index.html">WEB⁢/a>⁢/h1>
            <input id="night_day" type="button" value="night" onclick="
                if(document.querySelector('#night_day').value === 'night'){
                    document.querySelector('body').style.backgroundColor = 'black';
                    document.querySelector('body').style.color = 'white';
                    document.querySelector('#night_day').value = 'day';
                } else {
                document.querySelector('body').style.backgroundColor = 'white';
                document.querySelector('body').style.color = 'black';
                document.querySelector('#night_day').value = 'night';
                }
             ">
        </body>
  • document.querySelector('#night_day')
if 뒤에 나오는 이 부분은 사실상 자기 자신을 가리키고 있다. onclick 이벤트가 속해있는 자신이다 그러면 저 코드들을 어떻게 바꾸냐. this로 바꾼다 그러면 더이상 이 index 태그를 참조하는 것은 this로 해도 되기 때문에 'id="night_day'는 더이상 쓰지 않는다 그러면 얘를 지운다. 

        <body>
            <h1>⁢a href="index.html">WEB⁢/a>⁢/h1>
            <input type="button" value="night" onclick="
                if(this.value === 'night'){
                    document.querySelector('body').style.backgroundColor = 'black';
                    document.querySelector('body').style.color = 'white';
                    this.value = 'day';
                } else {
                document.querySelector('body').style.backgroundColor = 'white';
                document.querySelector('body').style.color = 'black';
                this.value = 'night';
                }
             ">
        </body>

코드가 효율적이 되었다. 유지보수하기 편리하다. 또 하나만 더 하자면,

  • document.querySelector('body')
이 부분이 계속해서 중복해서 등장하고 있다 프로그래밍 수업에서 어떻게 코딩을 잘하느냐. 코딩을 잘하는 방법을 알려드리면, 중복을 끝까지 쫒아가서 다 없애버리는 것이다 코딩을 하면서 여러 기술들을 쓰게 되는데, 여러 기술들이 출현하게 된 아주 결정적인 계기가 바로 중복의 제거와 관련되어있을 가능성이 크다

그래서 중복을 제거해보겠다.

  • var target = document.querySelector('body')
이렇게 적어놓으면 이제 target은 body 태그가 된다. 그리고 이것과 똑같은 내용을 무엇으로 바꾸나. target으로 바뀐다. 그러면 코드가 훨씬 간결해졌다 그리고 target이 가리키는, body를 다른 애로 바꾸면, 아래의 target이라고 적힌 애들 전체가 다 바뀌는 폭발적인 힘이 나타난다 중복되는 코드가 1억개라고 하면 그 힘을 더 잘 느낄 수 있다 변수를 활용하면 코딩을 하는데 큰 도움을 받을 수 있다
