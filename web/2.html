6. 기본문법 '태그'

본격적인 웹페이지를 이제 만들어보자. 기획상에서 그려놓았던 것을 한꺼번에 만들기 어렵기 때문에 조각내서 가자. 먼저 화면에 표시할 부분을 텍스트 문단이다.

[HTML 기본 문법]

  • TAG: 'strong'(열리는 태그), '/strong'(닫히는 태그)
옷에 붙이는 태그는 옷을 설명한다. strong 태그는 이건 중요한 부분이니 강조해야 할 부분이다. 라고 하는 것이다.(실제로 낯선 프로그래밍 용어가 나오면 사전을 찾아봐라. 그러면 우리에게 대체로 익숙한 뜻이 나올 것이다.)

컴퓨터 공학은 하나의 일종의 거대한 시이다. 왜? 물질을 기반으로 하지 않고, 논리적인 것을 기반으로 하기 때문에 컴퓨터공학으로 만든 것들을 사람들에게 설명해주고, 사람들이 마치 그게 손에 잡히는 것처럼 받아들이게 하기 위해 비유를 많이 사용한다. 만나는 개념들이 나올 때마다 시의 개념이라고 생각해봐라.

(1)-1. 굵은 글씨 강조
<b>web</b>
(1)-2. 밑줄(Under line)
<u>web</u>
(1)-3. 제목
<h1>W3C</h1>


7. 혁명적인 변화: [side study] 어려운 것과 쉬운 것

처음 배우는 것들은 가장 자주 사용되는 부품이면서도 완제품을 만들어내는 결합방법이다. 기초 문법을 배웠을 뿐이지만, 우리의 인생은 이것을 배우기 전과 배운 후로 나누어진다.

<실전 적용>

  • 웹사이트에서 오른쪽 마우스를 클릭하고 페이지 소스를 눌러보자.
  • 그러면 어떤 html 코드로 들어가있는지 알 수 있다.
  • 그리고 웹사이트의 어떤 궁금한 부분을 오른쪽 클릭해 검사를 누르면 어떤 코드인지 구체적으로 알 수 있다.

이제 내가 무엇을 모르는지는 아는 상태가 되는 것이다. 이제 나는 검색할 수 있고, 질문할 수 있고, 대화할 수 있고, 궁금해하기 시작한다. (혁명적인 순간이다.)

8. 통계에 기반한 학습

통계를 기반으로 해서 공부하는 방법에 대해서 이야기하자. html은 150개 이상의 태그가 존재한다. 150개를 그럼 다 외워야할까? 그렇지 않다. 태그를 배우고 이해했다면 사실상 모든 태그를 알고 있는 것과 다르지 않다. (1분이면 찾을 수 있기 때문) 태그를 하나도 모르고 있으면 너무 불편하다. 어떤 태그들은 지식이 있어야지 이해할 수 있는 부분들이 있다. 통계를 보면서 어떤 방향으로 공부해야할지 감을 잡아보자. (기본적으로 밑천이 되는 태그가 무엇인지를 스스로 판단하도록)

참고사이트 overview에 그래프가 있는데, 이 그래프는 전세계 수많은 웹페이지들이 몇 개 정도의 태그가 있는지 통계로 보여준다. 약 25-26개 정도 태그를 가지고 있는 웹페이지가 많다. 이미 8개의 태그를 알고 있다. (26개 이상 알고 있으면 평균 이상 간다는 얘기이다.) 여러 태그들이 웹페이지 상에서 사용되는 랭킹도 있다.

  • 사용 1위 ~ 22위: head / body / html / title / meta / div / a / script / link / img / p / span / li / ul / br / style / hi / h2 / input / form / strong /
가장 기본이 되는 것들은 적지만 이것을 응용해서 정말 많은 것들을 만들어내게 될 것이다 어떤 것을 공부할 때 항상 통계를 기반으로 무엇을 공부할까를 판단했으면 좋겠다. 그리고 처음 시작할 때, 자기가 가야할 길이 멀다고 해서 의기소침하지 않았으면 좋겠다.

9. 줄바꿈: br vs p (인기있는 태그 2가지 학습)

이 두 태그는 서로 경쟁관계에 있다. (어떤 경쟁관계에 있는가를 통해서 정보로서 웹이라는 관점에 대해 생각해보는 계기를 만들자) css를 지배하는 혁명적인 규칙을 접하는 기회도 될 것이다. 아까 썼던 vsCODE에서의 단락을 더 줄글을 길게 써보자. 실제로 웹상에서 띄워보니까 답답하게 보인다. 왜? 단략이 없기 때문 (코드에서 줄바꿈을 한다고 해도 실제로 달라지지 않았다.) 이제 줄바꿈을 하는 태그를 찾아야 한다.

  • 추천 검색어: html new line tag

(1) 줄바꿈

여백이 긴 문단을 이러한 코드로 인해 만들어낼 수도 있다. 두번 붙이면 문단이 나눠진다. br 태그는 줄바꿈이라는 시각적 의미만 가지고 있기 때문에 굳이 닫지 않는다.

(2) 단락 :

    추천 검색어: html paragraph tag
br과 p 중에 어떤 태그가 더 좋은 태그이냐? p가 더 좋은 결정이다. 왜냐하면 이 웹페이지를 정보로서 가치있게 만들기 때문이다. (저만큼이 단락이다 라는 것을 의미론적으로 표현이 가능. 그러나 br은 줄바꿈) 단점으로는 'br'을 여러번 쓴 것과 다르게 문단 간 간격에 제한이 있다. (그러나 우리에게는 css 기술이 있다!)

9-2. 실전 적용

</p><p style="margin-top; 40pc;">나는 꾸꾸입니다.
css 코드 중에 태그와 태그 사이에 여백을 나타내는 코드가 있다. 두번째 코드에다 style이라는 속성을 쓴다. css는 html이라는 완전히 새로운 언어의 문법이 들어오도록 약속했다. 태그의 사용을 통해 어디서부터 어디까지가 같은 단락이다. 라는 것을 표현할 수 있고, 시각적으로 부족한 부분은 css로 정교하게 제어가능하기에 의미에 맞는 태그를 사용하는 것이 좋은 방법이다.
