10. 그리드의 기본 사용법

본문을 분할된 오른쪽에 넣고 싶은데, 하는 여러 방법 중에 아주 최신의 방법인 그리드를 활용해보자 인류가 되게 오랜 시간 동안 정보들을 잘 배치하는, 레이아웃을 잘 짜는 방법을 코드를 통해 효율적으로 하는 것에 대해서 굉장히 어려움을 많이 겪었다 여러 가지 시행착오 끝에 plux라는 것도 등장하고, 궁극의 방법이라고 할 수 있는 그리드까지 도착했다 이번 시간에는 아주 최근에 등장한 기술인 그리드를 활용해서 목적을 달성해보겠다

그리드가 무엇인지 확인해보자 기본 파일을 열고, 두 개의 문자를 작성하겠다 


    <!doctype html>
<html>
<head>
    <title></title>
    <meta charset="utf-8">
</head>
<body>
    NAVIGATION ARTICLE
</body>
</html>
각각의 문자에 테두리를 보고 그것들을 나란히 배치해볼 것이다 그러려면 두 개 문자의 정보를 각각의 태그로 묶어주어야 한다 그런데 우리는 지금 단지 디자인을 하기 위해서, 저것들을 나란히 배치시키기 위해서 태그를 사용하려고 한다 이런 경우 예를 들어 h1 태그를 쓴다? 이상하다 h1 태그는 제목이라는 의미가 있다 디자인이라는 목적을 위해서 어떤 의미도 존재하지 않는 태그를 사용해야할 때가 있다 그럴 때 사용하라고 존재하는 무색무취와 같은 태그가 바로 div라는 태그이다 division의 약자이다 아무런 의미도 없고 단지 디자인의 용도로만 쓰는 태그를 감싸주면 보는 것처럼 두 개 문자가 나오게 된다 div 태그는 기본적으로 블럭 레벨 요소이기 때문에 화면 전체를 쓴다. 그래서 줄바꿈이 된다 

    <!doctype html>
<html>
<head>
    <title></title>
    <meta charset="utf-8">
</head>
<body>
    <div>NAVIGATION</div> 
    <div>ARTICLE</div>
</body>
</html>
똑같은 목적으로 쓰라고 고안된 태그가 있는데, span이다. 얘는 inline 요소이다. (inline이냐, block이냐를 따져서 쓰면 된다) 

    <!doctype html>
<html>
<head>
    <title></title>
    <meta charset="utf-8">
</head>
<body>
    <span>NAVIGATION</span> 
    <span>ARTICLE</span>
</body>
</html>
우리가 그리드를 알아보기 위해서는, 각각 태그의 부피감을 확실하게 알 필요가 있다 그래서 이 웹페이지에 있는 모든 div에 border:~~적어준다(부피감이 명확해지게) 

    <!doctype html>
<html>
<head>
    <meta charset="utf-8">    
    <title></title>
    <style>
        div {
            border:5px solid gray;

    </style>

    <span>NAVIGATION</span> 
    <span>ARTICLE</span>
</body>

부피감이 명확해질 것이다 두 개의 단어 요소를 나란히 놓기 위해서 최신 방법인 그리드를 쓸 것이다 그걸 위해서는 이 두 개의 태그를 감싸는 부모가 필요하다 부모 태그를 하나 만들 것이다 

    <!doctype html>
<html>
<head>
    <meta charset="utf-8">    
    <title></title>
    <style>

    </style>

<body>

     <div>ARTICLE <div>
</div>

</html>
그다음에 부모태그에 id를 부여할 것이다 

    <!doctype html>
<html>
<head>
    <meta charset="utf-8">    
    <title></title>

        div {
            border:5px solid gray;

'grid'라고 id 값을 정했지만 사실 아무거나 적어도 상관없다 

    <!doctype html>
<html>
<head>
    <meta charset="utf-8">    
    <title></title>
    <style>
        #grid {
            border:5px solid pink;
            }
        div {
            border:5px solid gray;
            }
    </style>
</head>

<div id="grid">
     <div>NAVIGATION <div> 
     <div>ARTICLE <div>
</div>
</body>

id값이 그리드인 태그에 대해서, pink를 주겠다. (이러면 이제 구분이 된다.)

내가 두 개의 태그를 나란히 배치하고 싶거나 하면, 그것을 감싸는 부모태그가 필요하다 그래서 디자인의 목적만으로 div id="grid" 라는 태그를 만든 것이다.

그러면 이제 준비가 끝났다. display를 활용하자. display는 블록 수준 요소, inline 또는 none을 통해서 어떤 태그가 표시되는 방법을 완전히 바꾸는 중요한 속성이었다 이 display 속성에 grid라고 하는 속성을 쓰면 어떻게 되는가. 아무런 변화가 없다 하지만 grid를 쓰고, 그다음에 grid-template : 우리는 두 개의 요소를 하나의 columns에 배치하고 싶다. 그러면 여기서 grid-template-columns:를 선택해서 첫번째 컬러는 150px 정도의 부피를 갖고, 뒤의 column은 나머지 공간에 다 쓴다. 그러면 이렇게 적는다 


        <!doctype html>
    <html>
    <head>
        <meta charset="utf-8">    
        <title></title>
        <style>
            #grid {
                border:5px solid pink;
                display:grid;
                grid-template-columns: 150px 1fr;
                }
            div {
                border:5px solid gray;
                }
        </style>
    </head>
    <body>
    <div id="grid">
         <div>NAVIGATION <div> 
         <div>ARTICLE <div>
    </div>
    </body>
    </html>
이 상태에서 내가 창을 크기조절하면 두번째 단어 영역은 창을 키우면 자동으로 커지고, 첫번째 단어 영역은 150px이라는 사이즈를 고정적으로 갖게 된다 그러면 1fr이 무엇이냐 

    <!doctype html>
<html>
<head>
    <meta charset="utf-8">    
    <title></title>
    <style>
        #grid {
            border:5px solid pink;
            display:grid;
            grid-template-columns: 1fr 1fr;
            }
        div {
            border:5px solid gray;
            }
    </style>
</head>
<body>
<div id="grid">
     <div>NAVIGATION <div> 
     <div>ARTICLE <div>
</div>
</body>
</html>
이러면 둘 다 같은 크기가 된다. 그리고 앞을 2fr로 하면, 화면 전체를 3fr로 했을 때, 앞은 2만큼, 뒤는 1만큼 화면 전체를 쓰기에 자동으로 조정되는 단위이다 150px을 사용하고, 뒤를 1fr을 쓰자

그렇게 하고, (grid를 사용하면 좋은 것이) 뒷 상자부분에 아무의미없는 텍스트를 넣어보자. 텍스트가 많아짐에 따라 자동으로 많아지고, 그에 따라 자동으로 왼쪽 상자도 커지게 된다 이것이 바로 grid이다 (최신 기술) 최신 css 기능을 활용하기 위해서는 지금 현재 사용해도 되는지 아닌지를 데이터에 근거해서 판단할 필요가 있다 이 사이트는 여러 css, html, js 중에서 현재 웹브라우저들이 얼마나 그 기술을 채택하고있는가에 대한 통계를 보여주는 것이다 어떤 곳에 접속한 환경에 따라 지원되는 기술인지 아닌지를 보여준다. 초록색이면 쓸 수 있다. 어디에서는 동작하지 않을 것이다라는 것을 알려준다. can i use.com

바깥쪽에 있는 div 태그를 그리드로 지정하기 위해서 id값으로 'grid'라고 지정할 것이다 그리고 순서는 다음과 같다.
  1. id가 grid인 태그에 대해 display를 grid로 지정
  2. columns(열)에다가 첫번째 열은 순서로 봤을 때 ol이다. 그러면 ol은 150px로 하고, div가 두번째 열인데, 이는 1fr로 한다
<html> <head> <meta charset="utf-8"> <title></title> <style> #grid { display:grid; grid-template-columns: 180px 1fr; } div { border:5px solid gray; } </style> </head> <body> <div id="grid"> <div>NAVIGATION <div> <div>ARTICLE <div> </div> </body> </html> 이제 검사해보자. 목차 블록의 패딩 값이 20px이다. 더블클릭하면 움직일 수 있다. 적당한 지점을 찾는다. 32px 정도 놓겠따. 그런 뒤 ol 태그에 대해서 padding-left:33px;한다

만약 본문을 조절하고 싶다면, 본문의 div 태그에 ⁢div id = "article">을 적는다 

    #grid {
        display: grid;
        grid-template-columns: 150px 1fr;
        }
    #article {
        padding:25px;
    }

이제 메인 style 속성에서 불필요한 값을 지워보자. 이를테면 ol 태그에 대한 width, margin 값은 불필요할 것 같다. (왜냐하면 목차 이외에 본문 내에서도 ol을 활용할 수 있기 때문) 그러면 (div id="grid") id 값이 grid인 태그 밑에 있는 ol 태그라고 하면 훨씬 의미가 분명해질 것이다라는 


    #grid ol {
        padding:25px;
    }

#grid #article {
