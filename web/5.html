19. 웹서버 운영하기

내 컴퓨터에 웹서버를 설치해서 남들이 내 컴퓨터의 웹페이지를 볼 수 있도록 해보자 잘 안될 수도 있다. 인터넷은 실제로 매우 복잡한 시스템이기 때문이다(제대로 작동하는 이유는 한가지이지만, 작동하지 않는 이유는 수십가지가 넘기 때문)

문제를 해결하기 위해 노력하고 싶다면 이것만 기억해보자. 우리가 하나의 문제를 해결하는데 많은 시간이 걸릴 수 있다 특히 중급자가 되면 그게 일상이 될 것이다 그러나 그 과정에서 우리는 많은 지식과 경험을 얻게 되고, 하나의 문제를 해결했지만 여러 개의 문제를 해결할 수 있는 능력을 갖게 된다고 생각한다

잘 풀리는 문제는 실력을 확장시켜주지만, 막히는 문제는 우리 문제를 확정시켜준다 우리는 확장과 확정 사이에서 끊임없이 진동해야 한다

웹서버 프로그램 설치하기

  • [질문 1] 웹브라우저는 제품명인가, 제품군인가
제품군이다
웹서버도 마찬가지이다. 웹서버라는 제품군이 있고, 여기에 속해있는 여러 가지 제품들이 있다.
  • Apache
  • IIS
  • Nginx
  • live Server : 이것을 활용할 것이다
이 서버는 VScode의 확장기능이다 2천 9백만회의 다운로드를 기록하고 있는, 아주 유명하고 잘나가는 웹서버이다
  1. VScode의 사이드 메뉴(왼쪽 메뉴바)에서 확장기능을 선택해라: extention(위에서 두번째)
  2. 'live server'를 검색한다
  3. 설치한다: 하단에 GO LIVE라고 뜨면 성공한 것이다
  4. GO LIVE 버튼을 클릭하면 라이브 서버가 시작된다: 잠시 뒤에 Port: 5500이라고 뜨면 웹서버가 실행된 것이다
  5. 동시에 웹브라우저가 뜰 것이다. 주소에는 'http://127.0.0.1:5500'이라고 적혀있을 것이다
  6. 지금 나는 내가 실행한 웹서버를 통해서 서비스되고 있는 웹페이지를 보고 있다
라이브 서버의 진짜 유용함은 '실시간 편집 기능'이다(이름도 라이브 서버기도 하다) 코드를 편집하면 실시간으로 웹브라우저의 내용이 변경된다. 리로드 할 필요가 없다면
  • 이번에는 웹서버를 꺼보자
  • 그리고 웹페이지를 리로드하면 웹사이트가 열리지 않는다
무슨 일이 벌어지고 있는지 보자


첫번째 주소는 웹브라우저가 파일을 직접 열었을 때 주소이다 두번째 주소는 웹서버를 통해서 접근했을 때의 주소이다 그렇기에 웹서버를 끄면 두번째 주소는 접근이 안 된다


잠깐 주소를 들여다보자

  • http: 하이퍼텍스트를 전송하는 프로토콜(통신규약)이라는 뜻이다
  • 127.0.0.1: 인터넷 프로토콜에서 사용하는 주소라는 뜻이다(ip라고 보통 부른다, ip주소는 43억개가 있는데, 그중에서 가장 특별한 주소가 바로 이 주소이다) 이 주소는 '내 컴퓨터 자신을 가리키자'라고 전세계적으로 약속한 주소이다 그래서 지금 우리의 웹브라우저는 같은 컴퓨터의 웹서버에 접속을 시도하고 있는 것이다(주로 개발할 때 많이 사용하는 주소이다)
  • port: 하나의 컴퓨터에 여러 개의 서버가 동작하고 있다고 해보자 그러면 웹브라우저는 서버가 여러개인데 누구랑 통신해야할까 그래서 각각의 서버는 각자의 포트 번호라는 것을 가지고 있다 그리고 지금 입력한 주소는 5500이니까 웹브라우저는 5500번 포트에 연결하고 있는 웹서버와 통신할 수 있게 되는 것이다
  • 그런데, 같은 컴퓨터 안에서 웹브라우저와 웹서버가 이렇게 서로 웹페이지를 주고받는 것은 현실세계의 웹과는 다르다 이걸 어디다 쓰겠는가(말하자면 혼잣말하고 있는 것이다)


현실의 웹은 어떻게 다를까
웹브라우저와 웹서버가 서로 다른 컴퓨터에 설치되어 있어서, 지구 반대편에 있는 컴퓨터도 마치 내 컴퓨터에 있는 문서인양 볼 수 있는 놀라운 일을 할 수 있는 것이다
이게 가능하게 하기 위해서는 방송국 같은 역할을 하는 '웹서버'가 필요하다 우리도 이렇게 해봐야 한다
이 작업을 하기 위해서는 컴퓨터가 두 대가 필요하다 핸드폰을 들어보자 휴대폰은 전화기의 탈을 쓴 컴퓨터이다 사실 전화기는 멸종되었다 휴대폰에 설치된 웹브라우저와 데스크탑에 설치된 웹서버가 서로 통신을 할 수 있다면 어떨까

  1. 우선 해야할 일이 있다. 컴퓨터와 휴대폰을 같은 네트워크에 연결한다. 같은 와이파이에 접속하자(그러면 두 개의 호스트가 같은 네트워크 안에 존재하게 된다)
  2. 또 하나 알아야 할 것은 내 컴퓨터의 IP Address를 알아내는 것이다 (컴퓨터의 주소를 따내는 것은 살짝 복잡하다. 운영체제마다 다르고.)
    • 선물이 있다. VScode의 확장기능 중 'ip address'라는 확장기능을 검색해서 설치하자
    • 그러면 화면 하단에 집모양 옆에 ip주소가 뜬다.(클릭하면 컴퓨터가 가진 다른 ip주소도 볼 수 있다. 이 주소를 쓰면 된다)
  3. 이제 우리가 알아낸 ip address를 휴대폰 인터넷 주소창에 입력한다
  4. 그러면 내가 만든 웹페이지가 뜬다
    5. 이렇게 되면 웹과 인터넷의 가장 큰 원리를 파악하게 된 것이다
