04. css의 기본 문법

웹페이지에 우리가 css를 포함시키기 위해서는 html과 css가 완전히 다른 언어기 때문에 웹브라우저로 하여금 어디서부터 어디까지가 css인지를 알려줘야 한다 그 방법은 바로 style태그이다.

이거 외에 한 가지 방법이 더 있는데, 태그를 통해 하는 것이 아니라 속성을 통해 하는 방법이다 우리 예제에서 현재 2번을 클릭하고 이곳에 있다라고 알기가 어려울 것이다 그래서 현재 사용자가 이곳에 머물고 있다는 뜻에서 2번을 파란색으로 표시하고 싶다 그러기 위해서는 css를 감싸고 있는 2번의 a 태그 안에다 직접 css 코드를 넣을 것이다 

     <li><a href="2.html" color:red>web</a></h1>
그러나 이렇게 하면, 웹브라우저 입장에서는 이것이 html인지 css인지 알 수가 없다 그래서 html과 css를 만든 사람들은 저 태그에 대해서 어떤 css에 효과를 주고 싶으면 이런 속성을 사용하도록 약속했다

바로 style이라는 속성이다 그 속성의 값으로 지정하면 이렇게 된다 

     <li><a href="2.html" style="color:red">web</a></h1>
style이라는 속성은 html과 css의 사용설명서에 적혀 있다 style이라는 속성을 쓰면, 그 속성의 값을 웹브라우저는 css의 문법에 따라 해석해서, 그 css의 문법에 따라 해석된 결과를 style 속성이 위치하고 있는 이 태그에게 적용할 것이다. 라고 적혀 있다

방금의 style이라는 속성은 html의 속성이다. 그리고 그 속성은 어떤 약속이 되어있냐 하면, 그 값으로 반드시 css의 효과가 들어온다. 이것이다 style 태그는 반대로, 효과만 있어서는 그 효과를 누구에게 지정할지를 설명할 수가 없기 때문에 a {}라고 하는 이런 코드가 추가적으로 더 필요한 것이다 이 추가 코드는 이 웹페이지에서 우리가 지금 주고 싶은 효과를 누구에게 줄 것인가를 선택한다는 점에서 '선택자'라고 부른다(영어로 selector) 선택자에게 지정될 효과(color:black;)를 효과라고도 하고, decoloration이라고 부른다('선언'이라는 의미) style 태그를 직접 사용한 경우에는 바로 저 style태그가 위치해있는 태그에 효과를 줄 것이기 때문에 선택자를 사용할 필요가 없다는 것도 기억하자

웹페이지 안에 css를 사용하는 두 가지 방법

  1. style 태그를 쓴다
  2. style 속성을 쓴다

조금만 더 꼬아보자. 웹페이지에서 보니까 밑줄이 좀 보기 싫다. 밑줄을 없애고 싶다 그러면 이렇게 할 수 있다 


        <style>
            a {
            color:red;
            text-decoration: none;
            }
        </style>
이렇게 적으면 웹페이지에 있는 모든 a 태그에 대해서 밑줄(장식)이 없어지게 된다 (하나의 description이 시작되고 끝나는 것을 구분하는 구분자로 ;를 적기 때문에 끝나면 꼭 ;를 적어주자. 그러면 다른 것들이 한 줄에 있어도 잘 동작한다)

만약 현재 있는 페이지의 링크 제목에만 밑줄을 넣고 싶다면? 

     <li><a href="2.html" style="color:red; text-decoration:underline">web</a></h1>


05. 혁명적 변화

a라는 부분은 이 웹페이지의 모든 a 태그를 선택한다는 점에서 '선택자(selector)'라고 부른다 color:red;라는 부분은 선택자가 지정하는 태그들에 대해서 어떠한 효과를 줄 것인지에 해당되는 것이다 color는 속성이라고 부르고, red는 속성의 값이라고 부른다 이것을 이해한다는 것은 큰 의미이다. 이제 내가 배우지 않은 선택자, 모르는 효과를 만나더라도, 효과가 필요하더라도 무엇을 모르는지를 아는 상태가 된다 무엇을 모르는지를 알면 무엇을 할 수 있나. 검색할 수 있고, 질문할 수 있고 궁금해할 수 있다

이제 두 가지의 여정을 걷게 된다 하나는 css를 활용해서 웹페이지를 디자인하는 어떠한 property가 존재하는가라는 것을 알아가는 과정에서 하나는 그 효과를 더 정확하게 선택해서 지정하기 위해서, 다양한 선택자를 알아가는 여정을 갖게 될 것이다


06. css 속성을 스스로 알아내기

어떤 불만족이 생기냐면, 이 웹이라는 정보는 웹페이지에서 제목인데, 뭔가 좀 더 컸으면 좋겠다. 그리고 가운데 정렬하고 싶다라는 생각이 든다 그러면 자연스럽게 이 web이라는 텍스트가 웹페이지에서 어떤 태그인지를 찾아야 한다(h1태그이다) 그러면 웹페이지에 있는 h1 태그 밑에 있는 텍스트를 더 크게 하고 싶다. 이런 마음을 갖게 될 것이다 웹페이지에 있는 모든 h1 태그는 이렇게 하면 된다고 


            <style>
                h1 {
            
                }
            </style>
웹이라는 텍스트를 크게 만들면 된다 해낼 수 있다. 개념을 아니까, 무엇을 모르는지 아니까 해결할 수 있다 검색어를 추천하겠다 직접 검색해서 문제를 해결해보자 우리가 하고 싶은 것은 css로 웹페이지를 디자인하는 것이니 css, 우리가 하려고 하는 것은 글씨의 크기니까 text, size이다. 그리고 어떤 속성을 이용하면 되는가. css, text, size, property 정도 검색해보자.

검색해서 보니, text가 아니라 'font'를 쓰고, size가 적혀있음을 알 수 있다. 그리고 뒤에 %가 되어있다 


    <style>
        h1 {
        font-size:60px
        }
    </style>
60px(픽셀)을 적용해보고, 60이 이만하다는 것을 보았으면, 저걸 기준으로 해서 적당히 크기를 찾으면 된다.이

그다음에 또 하고 싶은 것이 가운데 정렬이다 css, text, center, property 정도 검색해보자.(경험해보면 align이라는 용어도 활용할 줄 알게 될 것이다) 


        <style>
            h1 {
            font-size:60px
            text-align:center;
            }
        </style>


07. css 선택자의 기본

css를 지배하는 중요한 두 가지 토대가 있다. 1. 효과, 2. 선택자 이번 시간에는 선택자에 대한 중대한 토대를 닦자 예제를 변경해가면서 선택자를 하나하나 배워보자

    웹페이지에 있는 모든 a 태그를 기본적으로 검은색으로 바꾼다 사용자가 방문한 적이 있는 페이지 목차링크는 회색으로 바꾸고 싶다 현재 사용자가 머물고 있는 링크는 현재 방문하고 있다는 점에서 빨간색으로 표현하고 싶다 

    <style>
        a {
        color:black;
            }
    </style>
이중에서 우리는 현재 머물고 있는 페이지가 css 페이지이다 css 페이지 이전에 있었던 html과 현재 페이지인 css를 회색으로 표시해서 사용자에게 정보를 준다 (지금까지 봤던 정보에 대한 정보) 지금까지 배운 범위에서는 이렇게 처리했어야 했다. 스타일 태그를 인라인으로 적는다. 

       <body>
           <h1><a href="index.html">WEB</a></h1>
           <ol>
           <li><a href="1.html" style="color:gray;">HTML</a></li>
           <li><a href="2.html" style="color:gray;">CSS</a></li>
           <li><a href="3.html">JavaScrip</a></li>
           </ol>
그러면 하고자 하는 게 이뤄진다 그러나 이 경우 우리가 굉장히 불쾌하게 생각하는 중복이 발생하고 있다 이 중복을 없애는 방법은 우리가 지금 보고 있는 중복되는 코드를 지우자 html과 css라는 두 개의 텍스트를 같은 그룹으로 묶고, 그 그룹에 대해서 폰트 컬러를 회색으로 준다
2개에 대해서 'class'라는 html의 속성을 준다 

    <body>
        <h1><a href="index.html">WEB</a></h1>
        <ol>
        <li><a href="1.html" class="saw">HTML</a></li>
        <li><a href="2.html" class="saw">CSS</a></li>
        <li><a href="3.html">JavaScrip</a></li>
        </ol>
'class=""' 이 부분은 html이다 'saw'도 마찬가지로 html이다. (여긴 css가 들어가는 게 아니다) 즉 saw라고 하는 클래스의 값을 가지고 있는 태그를 두 개 만들었고, 우리가 하고 싶은 것은 이 웹페이지에 있는 모든 태그 중에서 saw라는 클래스 값을 가지고 있는 모든 태그에 대해서 폰트 컬러를 그레이라고 주고 싶은 상태이다 

    <style>
        a {
        color:black;
            }
        saw {
            color:gray;
        }
    </style>
적용해보자. 안된다. 왜 그럴까? 그 이유는 이렇게 우리가 'saw'라고 쓰게 되면, 'saw'라고 하는 것은, 이 웹페이지에 있는 모든 'saw'라는 이름의 태그를 선택하는 선택자이기 때문이다 우리가 하고 싶은 것은, '클래스의 값이 saw인' 태그인데, 그때 사용하는 약속되어있는 특수한 기호가 바로 .이다 saw 앞에 .을 붙이는 순간에 저것은, 저 선택자는 어떤 뜻이냐. 저 웹페이지에 있는 모든 클래스가 saw인 태그를 가르키는 선택자가 된다

class라고 하는 것은 어떤 의미가 있나? 우리가 학교에서 '반'을 클래스라고 한다 그 클래스라고 하는 이유는 같은 이유로 묶여진 학생들을 우리가 그룹핑할때 쓰는 말이다. 클래스라는 말에는 '그룹핑하다', '하나로 묶는다' 이런 뜻이 포함되어 있다.

2번 css 목차링크만을 빨간색으로 표시하고 싶다면 이렇게 표시하자 


        <style>
            a {
            color:black;
                }
            .saw {
                color:gray;
            }
            .active {
                color:red;
            }
        </style>
        

        <body>
            <h1><a href="index.html">WEB</a></h1>
            <ol>
            <li><a href="1.html" class="saw">HTML</a></li>
            <li><a href="2.html" class="saw active">CSS</a></li>
            <li><a href="3.html">JavaScrip</a></li>
            </ol>
이걸 통해서 알 수 있는 것은 class라는 속성의 값은
  1. 여러 개의 값이 들어올 수 있다.
  2. 띄어쓰기로 구분된다
  3. 하나의 태그에는 여러 개의 속성이 들어올 수 있고, 여러 개의 선택자를 통해서 하나의 태그를 공동으로 제어할 수 있다

그러나 이 방법은 좋은 방법이 아니다 a 태그는 2개 class의 영향을 받고 있다 그러나 a 태그는 왜 빨간색이 된 것일까? 그것은 순서 때문이다(active가 saw보다 나중에 등장했기 때문) 


        <style>
            a {
            color:black;
                }
            .active {
                color:red;
            }
            .saw {
                color:gray;
            }
        </style>
이렇게 순서를 바꾸면 회색이 된다 보다 가까이에 있는 명령이 더 큰 영향력을 갖는다 말하자면 권력이 같으면 가장 최근에 명령한 사람 말을 들어야 한다 좀 더 우선순위가 높은 것을 사용할 필요가 있다 그게 바로 'id' 선택자이다 


        <body>
            <h1><a href="index.html">WEB</a></h1>
            <ol>
            <li><a href="1.html" class="saw">HTML</a></li>
            <li><a href="2.html" class="saw" id="active">CSS</a></li>
            <li><a href="3.html">JavaScrip</a></li>
            </ol>
active를 컷해서 id로 옮겼다 지금 하고자 하는 것은 웹페이지에 있는 id값이 active인 태그를 선택하려고 한다 그러면 이렇게 하자 

    <style>
        a {
        color:black;
            }
        #active {
            color:red;
        }
        .saw {
            color:gray;
        }
    </style>
#active가 .saw보다 뒤에 등장했음에도 리로드를 했을 때 붉은 색이 된다
  1. 즉 id 선택자와 class 선택자가 붙으면, id 선택자가 이긴다는 것이다
  2. class 선택자와 a {} 같은 태그 선택자와 붙으면 class 선택자가 이긴다
  3. 그리고 만약 모든 선택자가 같은 수준에 있다면, 맨 마지막에 있는 선택자가 우선순위를 갖는다

이유를 알면 까먹지 않는다 왜 tag 선택자와 class 선택자와 id 선택자가 있다면, id 선택자가 가장 강하고, 그다음이 class, tag 선택자인가 이런 의문을 가져야 한다 id 선택자의 id값이 active인 태그가 이 웹페이지에서 1번만 등장하면, 그다음 active라고 하는 값은 쓰면 안 된다 써도 웹페이지가 동작을 안하진 않지만 쓰지 말라고 되어있다 즉, id의 값은 단 한번만 등장해야 한다(학교의 id는 학번이다, 대한민국의 주민등록번호) 이 핵심은, id는 중복되면 안 된다이다. 즉 active를 썼기 때문에 다른 곳에서는 active를 쓰면 안 된다(이 문서에서는 유일무이하다)

반면, 태그 선택자는 id 선택자보다 훨씬 더 포괄적이다 그래서 css를 만든 사람들은, 좀 더 구체적인 것이 포괄적인 것보다 우선순위를 높였다 그래야지만 태그 선택자를 통해서 전체적인 태그의 디자인을 해내고, 예외적인 것들의 id를 찍어가면서 예외를 두는 것이 디자인과 코딩하는데 훨씬 효율적이다 그리고 태그 선택자와 id 선택자 사이에 class 선택자라는 중간 정도 우선순위의 선택자가 있다 클래스 선택자는 tag 선택자보다는 높고, id 선택자보다는 우선순위가 낮다. 라는 것을 기억하자 참고 사이트

우리가 했던 것은 element selector이다(예제에 보면 p라고 적혀 있다) p라고만 선택자를 쓰면 그것은 element이다(=tag와 같은 뜻) 즉 모든 p 태그를 선택한다라는 뜻이었다

그리고 #id(id 선택자)도 배웠다 '#~~'라고 적으면, id값이 ~~인 태그를 선택한다라는 뜻이다

class 선택자는 '.~~'라고 적고, ~~클래스를 갖고 있는 모든 요소들을 선택한다

이외에도 여러 가지 형태의 선택자들이 있고, 이러한 선택자들을 조합해서 내가 원하는 효과를 정교하게 태그에 타겟팅할 수 있고, 내가 작성한 코드의 양을 획기적으로 줄일 수 있고 동시에 어떤 한 부분만 수정하면 나머지가 동시에 바뀌는 아주 똑똑한 코드를 짤 수 있는 토대가 마련이 된 것이다
