'생활코딩' 학습기록: CSS 1


01. 수업에 들어가며

웹이 처음 세상에 등장했을 때는 단 하나의 기술이 있었다. 바로 html이다. html을 이용하면 전자문서를 만들 수 있었다. 즉 컴퓨터를 통해서 정보를 표현할 수 있게 된 것이다 또 이 정보를 인터넷을 통해서 전세계 누구나 볼 수 있게 할 수 있다는 것은 당시로서는 혁명적인 사건이었다 자연스럽게 사람들은 웹에 열광하게 된다 하지만 사람들의 욕심은 끝이 없다 처음에는 html을 활용해서 문서를 만드는 것만으로도 행복했지만 곧 여러 불편함이 쏟아져나왔다

수많은 관심사 중에 우리 수업의 관심은 웹페이지를 조금 더 아름다우면서 보기좋은 형태로 만드는 방법에 대한 불만족이다 왜 우리는 css가 등장할 수밖에 없었는지부터 시작해서 web1 수업에서 html만으로만 만들어진 웹페이지를 아름답게 꾸며가면서 아름다움에 대한 욕망들을 실현해갈 것이다 그 과정에서 css를 지배하는, 쉽지만 가며 중요한 원리들을 살펴볼 것이다 또, 매우 자주 사용되는 효과들을 통계에 근거해서 공부하게 될 것이다 아름다움을 위해 필요한 지식과 경험들을 스스로 탐구할 수 있는 에너지를 갖게 될 것이다


02. css 등장 이전의 상황

불만족 상황이 생겨나며 웹을 만든 사람들은 웹페이지를 아름답게 만드는 방법에 대한 깊은 고민에 빠진다 그러면서 두 가지 갈림길에 선다

  1. 하나의 길은 쉽지만 한계가 있는 방법이다
  2. 다른 하나는 어렵지만 근본적인 해결책이었다
질문이다 html이라는 이미 있는 언어에 디자인과 관련된 새로운 태그를 추가하는 것과, 디자인에 최적화된 완전하게 새로운 언어를 만들어내는 방법 중 어떤 것이 더 쉬울까? 새로운 태그를 추가하는 것이 더 쉽다. 이게 바로 쉬운 길이다 웹브라우저를 만든 사람들은 쉬운 길을 택한다. 그러나 그 한계를 깨닫고 곧 어렵지만 근본적인 해결책을 찾는다 그것이 바로 css라는 새로운 언어이다 css를 보기 전에 쉽지만 한계가 있었던 방법을 살펴보겠다

지금은 사용되지 않는, 디자인을 위한 태그를 설명하겠다. (수업이 끝나면 지워버려도 된다. 지금은 쓰지 않는다) font라는 태그이다 선택한 정보들을 빨간 색으로 바꾸고 싶다. 그러기 위해서 필요한 태그가 font태그이다 


        <h1><a href="index.html">꾸꾸의 학습기록</a></h1>
        <ol>
            <li><a href="1.html"><font color="red"></font>web</font><a><h1>
        </ol>
그전까지는 웹페이지는 검은색 문자만 가능했는데, font 태그가 등장하면서 다른 색의 컬러를 표현할 수 있게 된다 사람들이 얼마나 행복했을지 생각해보자. 이전엔 이렇게 색깔을 바꿀 수 있는 것만으로도 행복했다 웹페이지가 많아지고, 웹기술이 중요해짐에 따라 이런 방식의 한계를 느끼게 된다. 이 한계를 잘 느껴야지 css의 본질을 알 수 있다

a 태그는 font 태그와 다르다, h1태그도 font태그와는 다르다.

  • 어떤 점이 다르냐 하면, a는 web이라는 정보가 링크이다. 라는 것을 설명하는 정보이다
  • h1 태그는 web이라고 하는 문자가 이 웹페이지에서 제목이라고 하는 중요한 정보를 준다(저런 정보를 해석해서 검색엔진과 같은 시스템들이 필요한 정보를 찾아주는 것이다)
  • 반면, font 태그는 웹에 대해서 어떠한 정보도 가지고 있지 않다. 다만, 빨간색으로 표현해야한다는 디자인을 나타낼 뿐이다 디자인은 매우 중요한 것이지만, 디자인 자체가 정보라고 할 수는 없다(시각장애인이 봤을 떄 정보가 아닌 것은 정보가 아니라고 판단할 수 있다)
웹페이지라는 소중한 정보 안에 정보가 아니라고 할 수 있는 디자인에 대한 코드가 섞이면서 웹페이지가 정보로서의 가치가 현격하게 떨어지는 문제를 갖게 된다

문제가 또 하나 더 있다. 극단적인 상상력을 활용해보자. a태그가 1억개가 있다고 생각해보자. 그런데 아침점심 저녁으로 하루는 빨간색으로 하라고, 그다음에는 들어본 적도 없는 파우더 블루로 바꾸라고 매일같이 요청한다면? 굉장히 불행하다. 불행감은 공부를 지속하는데 있어 굉장히 중요한 에너지원이다 충분히 절망했다면 이 상황을 극복하게 해주는 놀라운 기술인 css를 배웠을 때 해방된 기분이다. 그만큼 css가 귀하게 느껴질 것이다


03. css의 등장

웹브라우저가 처음나왔을 때 웹브라우저는, 어떤 컴퓨터 언어만을 해석해서 처리하는 프로그래밍이었을까? html이다 그래서 웹브라우저는 기본적으로 이 코드를 html이라고 생각한다 css라는 언어는 배우지 않았지만, css는 html과는 완전히 다른 컴퓨터 언어이다 그러면 웹브라우저에게, '웹브라우저야, 지금부터 내가 css 코드를 쓸테니까. 너는 이 코드를 html이 아니라 css의 문법에 따라 해석해야 돼'라고 html의 문법으로 그 얘기를 해주어야 한다

style이라는 태그는 html의 문법이면서 동시에 이런 뜻이다. '웹브라우저야, style태그의 안쪽에 있는 내용은 css니까 너는 이걸 css라는 언어의 문법에 맞게 해석해서 처리해야 돼'라는 의미이다 깜박거리는 곳에 css를 작성한다 원래 있던 코드가 원래 하고자 했던 것이 무엇이었나. 모든 a태그의 폰트 컬러를 붉은 색으로 해라. 라는 뜻이었다 그것을 css라는 새로운 언어의 문법에 따라 한다 (지금 하는 얘기가 css 전체를 지배하는 가장 중요한 얘기이다. 집중하자. 얼마나 효율적으로 기호가 만들어졌는지도 음미하자)
웹브라우저야, 여기 있는 모든 a태그에 대해서라는 뜻은 이것이다 


<style>
    a {

    }
</style>
font 컬러는 붉은 색으로 해줘 

    <style>
        a {
        color:red;
        }
    </style>
아까의 font태그와 똑같은 결과이다 결과는 동일하지만, 우리가 이 결과에 도달하기 위해 작성한 코드는 예전에는 html의 태그였고, 이번에는 css라는 새로운 언어의 문법이었다 둘을 놓고 봤을 때 뭐가 공부하기 어려울까. css는 학습자에게 부담이다 무엇이 더 만들기 쉬울까. 웹브라우저 입장에서는 font 태그는 그냥 태그만 추가하면 된다. 그러나 css는 새로운 언어를 해석하는 기능을 탑재해야 하기 때문에 굉장히 어려운 일이다 그럼에도 불구하고 css를 사용하는 이유는 css를 사용할 때 그런 것을 뛰어넘는 폭발적인 효과가 있기 때문이다

만약 이 a라는 폰트태그의 색을 검은색으로 모두 바꿔버리고 싶은데, 1억개가 있다면? 1억개를 다 바꿔야 한다 그러나 css에서는 이렇게 black으로 바꾸는 순간에 1억개든 10억개든 순식간에 바뀌는 폭발적인 효과를 갖게 된다 


    <style>
        a {
        color:black;
        }
    </style>
늘 코딩을 배우면서 유념할 점은 코딩을 어떻게 잘할까가 아니라 코딩을 어떻게 할까, 이다 유일하게 잘하는 방법은 중복의 제거이다 중복이 1억개면 효과는 훨씬 클 것이다 중복의 제거를 하면 어떤 장점이 생기는가. 웹페이지의 사이즈는 커질 것이고, 한두번 정도 하루에 사용자가 접속한다고 하면 필요없다 그러나 거대 기업이라고 한다면 중복된 코드가 있느냐 없느냐에 따라 하루에 1억씩 차이가 난다 규모가 커지면 중요한 문제가 된다 그리고, 여기있는 font태그가 똑같은 태그라고 해서 일괄적으로 확 바꿔버리면 저 태그를 작성했던 사람의 의도와 다르게, 바꾸면 안되는 부분조차 바뀔 수 있는 가능성이 있다(그러면 수정이 힘들어진다) 누군가 그러나 style이라고 써놨다면, 여기있는 black을 red로 바꾸는 것은 같은 성격의 것을 바꾸는 것이라고 상당히 높은 확률로 확신할 수 있다 이 웹페이지를 유지하고 보수하는 것을 훨씬 편리하게 할 수 있다. 가독성도 훨씬 높아질 것이다 웹페이지를 디자인하는 것이 기존의 html을 이용해서 디자인하는 것보다 훨씬 더 효과적으로 디자인할 수 있다라는 것이 우리가 css를 사용하는 매우 중요한 이유라는 것을 기억하면 좋을 것 같다 또 하나 중요한 것은, 이 웹페이지의 코드는 이전의 코드와 비교했을 때 디자인과 관련된 코드는 한줄도 들어있지 않다. 대신에, 디자인과 관련된 코드는 style 태그 안에 갇혀있게 된다 웹페이지를 해석하는 기계들은 디자인과 관련된 것들이 필요없다면 이 부분은 무시하고 이 정보만을 가지고 있는 코드를 분석해서 여러 가지 작업을 각자의 의도에 맞게 할 수 있다

이것이 css가 도입된 정말 중요한 이유이다

  • html은 너무나 중요하기 때문에, html이 정보에 전념하게 하기 위해서 html로부터 디자인에 대한 기능을 빼서 온 것이 바로 css이다
  • css를 통헤 웹페이지를 디자인하는 것이 html을 통해 디자인하는 것보다 훨씬 더 효율적이기 때문에 css라는 언어를 도입하게 되었다


(노마드) 추가 설명

css(Cascading Style Sheets)의 설명란을 보면, 직관적으로 알 수 있는 부분이 있다. 그것은 바로 css는 html과 같이 써야한다는 것이다. 둘중에 하나씩만 사용할 수 없다. 두 가지는 같이 사용한다.

css는 browser에게 웹사이트가 어떻게 보여야 하는지에 대해 알려준다. 이게 전부이다. html이 '브라우저야. 이건 title이야.'라고 한다면, css는 ' 브라우저야. 그 title은 녹색이어야 해'라고 하는 것이다. html이 뼈대라면, css는 근육이다. css가 있으면 특정 신체부위를 더 크게나 작게 만들 수 있다.
