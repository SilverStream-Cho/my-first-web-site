20. 배열

지금부터 배열이라는 주제의 이야기를 시작하겠다 배열은, 이전에 코딩 프로그래밍을 처음 시작했을 때를 생각해보면, 처음 도전했을 때는 실패했다 실패한 곳이 배열이다. 그 이유는 이게 도대체 왜 필요한가였던 것 같다 배열이 왜 필요한지는 배열 자체로는 알기가 힘들다. 이 수업 다음 수업인 '반복문'에서 배열이 왜 필요한지 알 수 있을 것이다 반복문을 이용해서 웹 애플리케이션을 만드는 과정을 보게 되는 그 이후에, 조금 더 실용적인 예제가 나올테니 참아보자 이번 시간에는 조금 관념적인 이야기를 할 것이다

비유로 출발해보자. 집에 시간이 지날수록 살림이 늘어난다 살림이 늘어나면 마음이 힘들어지고, 뭘 사게 되나? 책장이나 또는 수납상자나, 집을 늘릴 수도 있을 것이다. 방이 여러 개 있는 집으로. 이런 식으로 집에 있는 사물이 많아질수록 그 사물들을 잘 정리정돈할 필요성이 생기고, 그 필요성에 따라가면서 정리정돈할 수 있는 여러 수납공간을 마련하게 된다. 코딩 또는 프로그래밍도 마찬가지이다 프로그래밍을 하게 되면 굉장히 많은 데이터를 다루게 된다 그 데이터들은 제각기 성격들이 다르다 데이터들이 많아짐에 따라 그냥 코딩을 하는 것에 따라서는 너무 복잡해서 인간의 한계에 도달하면서, 프로그램을 더 복잡하게 하지 못하는 순간에 도달한다 사람이 감당할 수 없을만큼 복잡해진 소프트웨어는 사람이 감당할 수 없는 소프트웨어가 된 것이다

그러면 어떻게 해야 하나. 집으로 돌아오면 우리는 수납상자를 통해서 음식들을, 사물들을, 수납상자나 냉장고 같은 곳에 넣고, 그다음부터는 수납상자 하나만 생각하면 된다. 냉장고 하나만 생각하거나. 즉 우리 머릿속에 다시 단순해지는 효과를 갖게 된다

배열도 일단 마찬가지의 느낌으로 상상해보자 데이터가 많아짐에 따라 많은 데이터를 그냥 둘 수 없기 때문에 데이터 중에 서로 연관된 데이터를 잘 정리정돈해서 담아두는 일종의 수납상자, 그것이 바로 배열, 영어로는 array 라고 생각하자. 이번 시간에는 배열의 문법을 살펴볼 것이고, 여러 성격들을 살펴볼 것이다(실용적인 얘기는 나오지 않는다) 


            <!doctype html>
            <html>
            <head>
            <meta charset="utf-8">
                <title></title>
                <body>
                <h1>Array</h1>
            </body>
            </html>
첫번째로 살펴볼 것은 배열의 방법이다 (syntax는 문법이라는 뜻이다) script 태그로 문법을 적어보자 

        <!doctype html>
        <html>
        <head>
        <meta charset="utf-8">
            <title></title>
            <body>
            <h1>Array</h1>
            <h2>Syntax</h2>
                <script>
                    ["egoing", "leezche"];
                </script>
        </body>
        </html>
문자는 어떤 기호로 끝나서 어떻게 끝나나? 큰따옴표이다. 배열은 []로 끝낸다 이 안에다 값들을 적는데, 여러 개의 값들을 적을 수 있다 그리고 값과 값 사이는 이렇게 ,로 구분한다. 그러면 이렇게 배열을 만든 다음에 이 상태로는 쓰기가 불편하다 그래서 이 배열을 변수에 담을 것이다 

            <!doctype html>
            <html>
            <head>
            <meta charset="utf-8">
                <title></title>
                <body>
                <h1>Array</h1>
                <h2>Syntax</h2>
                    <script>
                       var coworkers = ["egoing", "leezche"];
                    </script>
            </body>
            </html>
coworkers라는 변수에, 배열이라고 하는 새로운 데이터타입이 담긴 것이다 그 데이터타입에 coworkers라는 이름을 붙였다고도 할 수 있겠다(변수를 썼다는 것은) 이건 새로운 수납상자를 가져다놓으면서 거기에 물건을 두 개 담은 것과 똑같다

그러면, 배열을 만드는 것을 해봤으니, 이번에는 꺼내보자(만들었으면, 꺼낼 줄도 알아야 한다) 배열을 가져오는 방법을 살펴보자 


            <!doctype html>
            <html>
            <head>
            <meta charset="utf-8">
                <title></title>
                <body>
                <h1>Array</h1>
                <h2>Syntax</h2>
                    <script>
                       var coworkers = ["egoing", "leezche"];
                    </script>
                <h2>get</h2>
                <script>
                    document.write(coworkers[0]);
                </script>
                    
            </body>
            </html>
꺼내오고 싶으면, 이렇게 하면 되는데, 이것의 출력값은 'egoing'이다. 즉, 첫번째 자리에 있는 값은 0번째라는 뜻이다. (이것을 index라고 한다) 그러면 index 1번은 leezche가 된다.

또 하나 핵심적인 것을 배워보자. 들어있는 값이 몇 개인가를 체크하는 것이다 검색을 통해 이것을 알아낼 수 있다 (검색어: JavaScript array count) 검색 결과가 배열을 만들고 거기서 .length;를 하면 몇 개의 값이 있는가를 알려주는 것 같다 이런 것을 참고해서 만들어보자 


                <body>
                <h1>Array</h1>
                <h2>Syntax</h2>
                    <script>
                       var coworkers = ["egoing", "leezche"];
                    </script>
                <h2>get</h2>
                    <script>
                        document.write(coworkers[0]);
                    </script>
                <h2>count</h2>
                    <script>
                        document.write(coworkers.length);
                    </script>
            </body>
            </html>
헷갈리지 말아야 할 점이 여기에 있다. 우리가 index를 정할 때는 0은 첫번째고, 1은 두번쨰가 된다 그러나 개수는 1부터 쓰기 때문에 값이 두개라면 length의 값은 2가 된다

데이터를 추가하는 방법도 살펴보자. 만약 데이터를 추가하고 싶다. 그러면 검색을 통해 알아보자 (검색어: JavaScript array add data) 


            <body>
            <h1>Array</h1>
            <h2>Syntax</h2>
                <script>
                   var coworkers = ["egoing", "leezche"];
                </script>

            <h2>get</h2>
                <script>
                    document.write(coworkers[0]);
                </script>

            <h2>add</h2>
                <script>
                    coworkers.push('duru');
                    coworkers.push('taeho');
                </script>
 
            <h2>count</h2>
                <script>
                    document.write(coworkers.length);
                </script>
        </body>
        </html>
이렇게 실행시키면, length는 4개의 값이 들어있기 때문에 숫자 4가 카운팅된다 배열은 push나 여러 가지 명령어가 있다. push는 데이터를 끝에 추가하는 것이었고, 데이터를 앞쪽에 추가하고 싶거나 데이터를 지우고 싶거나 중간에 낑겨넣고 싶거나, 여러 필요성이 있다. 바로 그런 필요성들을 다 처음부터 외울 필요는 없다. 검색할 수만 있으면 된다 (검색어: JavaScript array) 이 정도만 검색해도 여러 가지 기능을 이용해서 배열이라고 하는 수납상자에 데이터를 넣고, 빼고, 몇 개가 있는지를 알아내고, 그것을 가지고 작업을 처리하고 하는 것에 대한 정교한 기능들이 마련되어있기 때문에 어떤 기능들이 있는지 대충 쓱 눈으로 봐도 좋고, 나중에 코딩을 하면서 복잡한 상황에 맞닥뜨리게 되면, 자연스럽게 배열을 좀 더 복잡하게, 풍부하게 활용할 필요성을 느끼게 되는데, 검색을 통해 공부하면 된다 미리 처음부터 이 모든 것을 우겨넣을 필요는 없다

배열은 이거 자체로는 어떤 것에 쓰이는지 모른다. 다음 시간에 환상의 콤비인 '반복문'을 보면서 배열의 의미에 대해서 생각해보자


21. 반복문

반복문의 예제를 보면서 반복문의 문법을 살펴보겠다 반복문은 영어로 Loop라고 부른다 write를 쓰기 전에, 여기에 리스트를 만들 것이다 


            <body>
            <h1>Loop</h1>
                <ul>
                <script>
                    document.write('⁢li>1⁢/li>'); 
                    document.write('⁢li>2⁢/li>'); 
                    document.write('⁢li>3⁢/li>'); 
                    document.write('⁢li>4⁢/li>');   
                </script>
                </ul>
            </body>
이걸 한 이유는 순서대로 실행되는 어떤 기능들 하나하나라고 생각해주자. 1번째, 2번째, 3번째, 4번째 기능이 있다고 했을 때, 실행해보자 결과를 보면, 마치 ul과 li를 쓴 것처럼 숫자가 하나씩 한줄에 적혀있다

이렇게 순서대로 실행되는 것이 프로그램의 기본적인 실행순서이다 그런데, 어떤 경우에는 필요에 따라 저기에 있는 2번째와 3번째 실행되어야 하는 명령을, 여러 번 반복해서 실행해야 할 때가 있다 그런 경우에 우리가 할 수 있는 기본적인 방법은 반복적으로 코드를 짜는 것이다 이렇게 하면 문제는 만약 이 반복해야 할 2줄의 프로그램 코드가 1억개라면? 반복적으로 copy & paste하는 것이 불가능할 것이다 그렇게 그렇게 해서 힘들게 만들어놨더니 이 내용에서 버그가 있거나 수정사항이 있다면 1억개를 다 바꿔야 하는 절망적인 상황에 놓이게 된다 바로 그러한 상황에서 충분히 절망하고 있는 우리에게, 반복문이 나타난다면 반복문을 배우는 것따위의 어려움은 전혀 어렵지 않을 것이다 반복문을 통해서 무엇을 할 수 있는지, 반복문이 없었을 때 얼마나 절망스러울 수 있는지 경험하지 못한 상태에서는 반복문이 어려울 것이다 상상력을 통해서 공부할 심적인 마음을 잘 만들어주자

두번째와 세번째 코드만 세 번 반복하도록 하자 그러기 위해서 반복문의 기본문법 while을 쓸 것이다 (이것 외에 for문 같은 것이 있는데, while만 다룰 것이다) 


            <body>
            <h1>Loop</h1>
                <ul>
                <script>
                    document.write('⁢li>1⁢/li>'); 
                    while(true) {
                    document.write('⁢li>2⁢/li>'); 
                    document.write('⁢li>3⁢/li>'); 
                    }
                    document.write('⁢li>4⁢/li>');   
                </script>
                </ul>
            </body>
        </html>
while의 뒤를 보면 if문과 생긴 것이 비슷하다. if 뒤의 괄호에는 무엇이 들어갔는지 기억하는가? true나 false가 들어갔다 boolean 데이터타입이 들어간다. 여기도 마찬가지이다. 저기는 둘 중의 하나만 들어갈 수 있고, 안쪽이 true인 동안은 중괄호 안에 있는 내용이 반복적으로 실행된다(언제까지? 괄호 안의 내용이 false가 될 때까지)

반복적으로 실행되는 순서는 이렇다. 


            <body>
            <h1>Loop</h1>
                <ul>
                <script>
                    document.write('⁢li>1⁢/li>'); 
                    while(true) {
                    document.write('⁢li>2⁢/li>'); 
                    document.write('⁢li>3⁢/li>'); 
                    }
                    document.write('⁢li>4⁢/li>');   
                </script>
                </ul>
            </body>
        </html>
괄호 안에 true가 있을 때, javascript는 while문이 실행되면 괄호 안을 제일 먼저 본다 그리고 true면 중괄호 안에 있는 것을 하나하나 실행시켜서, 중괄호 안의 마지막까지 가면, 다시 이 값이 true인지, false인지 확인한다 이걸 반복하고, 괄호 안쪽이 false인 것을 확인하면 그제서야 중괄호 밖의 아래에 마저 남은 것을 실행시킨다 즉, 반복문은 순서대로 실행되는 프로그램의 실행순서의 흐름을 제어하는 제어문이라고 할 수 있다(if문, 조건문과 함께) 이렇게 실행시키면 웹브라우저가 컴퓨터 자원을 다 가져가버릴 것이다. 왜냐하면 무한히 반복되기 떄문에 그래서 반복문은 반복문을 언제 종료할 것인가를 잘 지정해주어야 한다

이 두 줄의 코드가 3번 반복되게 실행시키고 싶다. 그러려면 이 두줄의 코드가 몇 번 실행되었는지를 어딘가에 적어놓을 필요가 있다 


            <body>
            <h1>Loop</h1>
                <ul>
                <script>
                    document.write('⁢li>1⁢/li>'); 
                    var i = 0;
                    while() {
                    document.write('⁢li>2⁢/li>'); 
                    document.write('⁢li>3⁢/li>');
                    i = i + 1; 
                    }
                    document.write('⁢li>4⁢/li>');   
                </script>
                </ul>
            </body>
        </html>
그것을 하기 좋은 것이 변수이다(그리고 습관적, 관습적으로 'i'라는 변수에게 그런 역할을 시킨다) 그리고 반복문의 중괄호 안에 맨 마지막이 되면 i+1이 i의 값을 대신하도록 한다. i가 0부터 실행되어서 반복문이 실행될 때마다 i의 값이 1씩 증가할 것인데, 만약 3번 이 반복문을 실행시키고 싶다면? 

            <body>
            <h1>Loop</h1>
                <ul>
                <script>
                    document.write('⁢li>1⁢/li>'); 
                    var i = 0;
                    while(i ⁢ 3) {
                    document.write('⁢li>2⁢/li>'); 
                    document.write('⁢li>3⁢/li>');
                    i = i + 1; 
                    }
                    document.write('⁢li>4⁢/li>');   
                </script>
                </ul>
            </body>
        </html>
이러면 출력값은 1 2 3 2 3 2 3 4 하고, 2-3이 3번 등장하고 있다(i의 값이 2일 때 중괄호 마지막까지 가서 i의 값이 +1하여 3이 되면, 다시 위에서 반복하기 전에 확인했을 때 false값이 되어 반복이 바로 종료되고 4가 출력되는 코드로 넘어간다)


22. 배열과 반복문

지금부터 배열과 반복문을 합성해서 이 각각의 개념들이, 기능들이 서로 결합해서 어떤 강력한 시너지 효과를 내는지를 살펴보는 시간을 갖겠다 js 없이 문제를 만들어보겠다 


            <body>
            <h1>Loop & Array</h1>
            <h2>Co workers</h2>
            <ul>
                <li>egoing</li>
                <li>leezche</li>
                <li>duru</li>
                <li>taeho</li>
            </ul>   
        </body>
        </html>
만약 저기있는 글 목록이 1억개이고, li의 태그가 단순한 태그가 아니라 아주 복잡한 태그이고, 이것에 대한 수정이 아주 빈번하게 일어난다면? 이렇게 글 목록을 작성하는 것이 보통이 아닐 것이다 이렇게 서로 연관된 데이터들을 js에서 담는 방법이 무엇이었나? '배열'이었다. 그 배열에 담겨있는 데이터를 순차적으로 꺼내서 li라는 태그로 만들어주는 역할을 누가 하는가? '반복문'이다 지금부터 배열과 반복문을 하이브리드 해서, 직면한 문제를 해결해보겠다

script 태그를 만들겠다. 그리고 coworkers라고 하고, 배열을 만들 것이다 각각의 데이터를 배열에 넣을 것이다. 각각의 배열 항목들을 원소라고 한다. element 그러면 coworkers라는 변수가 가리키는 이 배열에 담겨있는 데이터를 하나하나 꺼내서 우리가 li 태그를 만들면 될 것이다 


            <body>
            <h1>Loop & Array</h1>
            <script>
                var coworkers = ["egoing", "leezche", "duru", "taeho"];
            </script>
이제 아래를 script 태그로 바꾼다 

            <body>
            <h1>Loop & Array</h1>

                var coworkers = ["egoing", "leezche", "duru", "taeho"];
            </script>
            <h2>Co workers</h2>

                <script>
                    document.write('<li></li>');
                </script>
            </ul>  
        li 사이에 각 데이터들을 넣으면서 하면 될 것이다
        이걸 하기 위해서는 4번에 걸쳐서 document.write를 해야 한다. 
        그러면 일단 이걸 4번 실행시키도록 해본다
        

            <body>
            <h1>Loop & Array</h1>
            <script>
                var coworkers = ["egoing", "leezche", "duru", "taeho"];
            </script>
            <h2>Co workers</h2>
            <ul>
                <script>
                    var i = 0;

                    document.write('<li></li>');

        항목 앞에 쓰일 태그가 4개 나온 것은 코드가 잘 나왔다는 뜻이다. 이제
        coworkers라는 변수 안의 배열에서 반복문이 실행될 때마다 하나하나의 원소값을 가져와서 여기 위치시키면 될 것이다
        재미있는 것은 i라는 변수를 우리가 가져왔고, 그 변수는 0부터 시작해서 3까지 1씩 증가한다. 
        그 말은 egoing은 0번째 index이고, leezche는 1번째 index이다. 우연의 일치인지..

        가져오고 싶은 데이터의 인덱스 값을 넣어준다
        그런데 아까 얘기했듯이 i값이 index값과 일치한다고 하는 특성을 활용할 수 있다는 것이다
        이렇게 해서 reload하면 원하는대로 나온다

                이 두 개의 기능은 아주 환상의 콤비같은 기능이라고 할 수 있다
                js가 가지고 있는 배열과 반복문 기능을 순수하게 (웹브라우저랑 상관없이) 살펴보았다
