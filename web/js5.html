24. 함수 예고

함수라는 도구의 사용법을 살펴보자. 함수는 영어로 function이라고 부른다. 함수는 떠오르는 이미지가 수납상자이다. 살다보면 수납상자를 장만해서 정리정돈을 한다. 마찬가지로 코드가 많아지면, 그 코드를 잘 정리정돈하기 위한 도구들이 필요하다. 첫번째로 생각할 아주 간단하면서도 가장 강력한 도구가 함수이다. 그것보다 조금 더 큰 도구는 '객체'라는 것도 있다. (이건 나중에 살펴보자)

input이라고 하는 코드가 하나 있다. js가 onclick안에 들어가 있다. 그런데 극단적으로 생각해서, 여기 있는 input 태그가 1억개가 있다고 해보자. (그래야 충분히 절망적이다) 이런 상황에서 1억개를 다 코딩해볼 수 없으니 2개만 복사해보자. 여기에서 만약에 powderblue라는 부분을 다른 색으로 바꿔야 한다? 그러면 아래의 1억개의 powderblue를 다 바꿔야 한다. 쉽지 않을 것이다. 일괄로 기계를 이용해서 바뀐다고 하더라도 바뀌면 안되는 powderblue까지 바뀔 수도 있다. 그리고 아래에 복붙한 코드가 우리가 보기에 위의 코드와 아래의 코드가 같은 코드라는 것을 확신하기도 어렵다. (눈으로 확인하기 힘들다. 어느 시점부터는 사람이 확인하기 불가능해진다) 그리고 코드의 양이 많아진다. 그러면 웹페이지의 크기가 커진다. 그러면 인터넷에 전송하기 위한 시간과 노력이 그만큼 들어간다는 것이다. 이 절망적인 상황에서 우리를 구원해주는 것이 함수이다. 함수가 어떻게 불행으로부터 탈출시켜주는지 확인해보자.

25. 함수의 기본

이번 시간에는 함수를 이론적으로 정리하는 시간을 갖도록 하겠다. 함수는 영어로는 function이라고 한다. 객체를 배우면 (똑같은 건데) method라고 부르는 경우도 있다. 그러나 이 말은 잊고 함수라는 말만 기억하자.

함수의 기본적인 문법을 배우고, 그다음에 파라미터와 아규먼트를 배운 다음, 리턴을 배울 것이다. 우선 함수의 기본적인 문법부터 배워보겠다. 순서대로 실행되는 것을 보기 위해 리스트를 만들겠다. 


        <!doctype html>
        <html>
        <head>
            <meta charset="utf-8">
            <title></title>
                <body>
                    <h1>function</h1>
                    <h2>Basic</h2>
                    <ul>
                        <script>
                            document.write('<li>1</li>');
                            document.write('<li>2-1</li>');
                            document.write('<li>2-2</li>');
                            document.write('<li>2-1</li>');
                            document.write('<li>2-2</li>');
                            document.write('<li>3</li>');
                        <script>
                    </ul>    
        </body>
        </html>
코딩을 하다보면, 이렇게 반복되어야 하는 경우가 있을 수 있다. 이런 경우라면 무엇을 생각할까. 이 부분을 뭐로 처리하려고 하겠는가? 반복문을 쓸 것이다. 그러나 반복문을 쓸 수 없는 경우가 분명 있다. (만약 연속적이지 않게 반복된다면?) 그러면 반복문을 쓰는 것이 어렵거나 불가능해진다. 그리고 항상 무언가를 하나가 아니라 두 개 이상을 선택한다면, 언제나 1억개라고 극단적으로 생각하자. 그러면 함수가 필요하다.

반복적으로 등장한 코드를 일단 복사한다. 그리고 함수를 만드려고 하는 것이기 때문에 '웹브라우저야, 지금부터 나는 함수를 만든다는 의미에서 function이라고 적을 거야' 한다. 그리고 만드려는 함수의 이름을 알려준다. 괄호는 그냥 적어본다. 중괄호에 아까 복사해둔 코드를 붙여넣는다. 


        <!doctype html>
        <html>
        <head>
            <meta charset="utf-8">
            <title></title>
                <body>
                    <h1>function</h1>
                    <h2>Basic</h2>
                    <ul>
                        <script>
                            function two(){
                                document.write('<li>2-1</li>');
                                document.write('<li>2-2</li>');
                            }
                            document.write('<li>1</li>');
                            two();
                            document.write('<li>3</li>');
                            two();
                        <script>
                    </ul>    
        </body>
        </html>
웹브라우저는 앞의 function을 보고, 뒤의 중괄호를 보고 나서 '아, 이 사람은 two라는 함수를 만드려고 하고 있구나, 그리고 그 함수의 내용은 여기 이것이구나.'라고 생각하고 기억하게 된다. 그리고 그렇게 기억하고 있다가. 뒤에 two라고 하면 거기까지는 모른다. 그러나 괄호가 나오고, 괄호가 닫히는 것을 보면, 함수는 '아, 이것은 two라는 함수의 이름이었고, 이 사람은 two라는 함수를 실행하고 싶구나' 라는 것을 알게 된다. 그래서 여기있는 이 코드들이 마치 여기 원래 있었던 것처럼 실행하게 된다. 이것이 바로 함수의 기본적인 문법이다. (너무 중요한 이야기이다.)


26. 함수: 매개변수(parameter)와 인자(argument)

우리가 지금까지 살펴보았던 함수는 어떤 형태의 함수라고도 볼 수 있냐면, 자판기이다. 자판기의 제품이 하나밖에 없어서 언제나 똑같은 제품을 받을 수 있는 자판기이다. 저것만으로도 우리에게 자판기는 편리함을 주지만, 좀 아쉽다. 원하는 제품을 선택하면, 자판기가 그 제품에 해당되는 구체적인 제품을 우리에게 제공해주는 자판기면 좀 더 효용이 있을 것이다. 그래서, 우리가 제품을 선택하는 것은 말하자면 입력이라고 할 수 있고, 자판기가 입력에 대해서 해당되는 제품을 제공하는 것을 출력이라고 하는데, 이는 자판기 뿐만이 아니라 이 세상의 모든 것들이 입력과 출력으로 이루어져 있는 것이라고 생각한다. 입력과 출력에 해당하는 것이 수학적으로는 함수에 해당하는 것들이다. 프로그래밍에서의 함수를 살펴보며, 함수가 어떤 효용이 있는지, 어떤 문법적으로 되어있는지 살펴보자.

우리 함수는 입력과 출력으로 이루어져 있는데, 바로 그 입력에 해당되는 것이 parameter, 매개변수이다. argument는 '인자'이다. 그리고 출력은 return이라는 것과 관련이 있다. 그다음에 전혀 실용적이지 않으면서 실용적인 척하는 코드를 쳐보겠다. 


        <!doctype html>
        <html>
        <head>
            <meta charset="utf-8">
            <title></title>
                <body>
                    <h1>function</h1>
                    <h2>Parameter & Argument</h2>
                        <script>
                                document.write('1+1');
                        <script>
        </body>
        </html>
코드가 일억개가 있다고 생각해보자. 아니면 엄청나게 복잡한 코드이다. 이것을 배운 바에 따르면 어떻게 함수화할 수 있을까? 

            <!doctype html>
            <html>
            <head>
                <meta charset="utf-8">
                <title></title>
                    <body>
                        <h1>function</h1>
                        <h2>Parameter & Argument</h2>
                            <script>
                                function onePlusOne(){
                                    document.write('1+1');
                                }
                                onePlusOne()
                            <script>
            </body>
            </html>
onePlusOne()라는 함수는 언제나 똑같이 동작하는 함수이다. 그런데 우리는, 이 함수를 실행할 떄, 어떤 입력값을 주면, 함수가 그 입력값에 따라서 다른 결과를 출력하게 한다면 멋질 것이다. 예를 들면, 여기서 sum(2, 3)이라고 하면 결과는 '5'가 된다. sum(3, 4)를 해서 결과가 '7'이 될 수 있다면? 사실 정말 복잡한 코드라면? 정말 복잡한 과정에서 만들어진 것이라면, sum이라는 단 하나의 함수로 입력값이 무엇이냐에 따라 다른 함수결과를 출력할 수 있게 된다면 멋질 것이다. 

                <!doctype html>
                <html>
                <head>
                    <meta charset="utf-8">
                    <title></title>
                        <body>
                            <h1>function</h1>
                            <h2>Parameter & Argument</h2>
                                <script>
                                    function sum(){
                                        document.write();
                                    }
                                    sum(2, 3);
                                    sum(3, 4);
                                <script>
                </body>
                </html>
2가 해당되는 괄호에 left, 3에 해당되는 두번째 자리에는 right라는 변수를 정의할 것이다. 바로 이 변수들이 한국어로는 매개변수, parameter이다. 만약 sum(2, 3) 코드를 실행하려면, 이때 left의 값은 2이고, right의 값은 3이라고 약속된 상태이다. 

                    <!doctype html>
                    <html>
                    <head>
                        <meta charset="utf-8">
                        <title></title>
                            <body>
                                <h1>function</h1>
                                <h2>Parameter & Argument</h2>
                                    <script>
                                        function sum(left, right){
                                            document.write(left+right+'<br>');
                                        }
                                        sum(2, 3);
                                        sum(3, 4);
                                    <script>
                    </body>
                    </html>
안에 있는 코드가 1억줄짜리라서, 쳐다도 보기 싫다고 하자. 우리는 단지 sum이라는 함수에 2, 3이라는 값을 입력해주는 것을 통해서 우리는 5라는 결과를 화면에 출력할 수 있게 된다. 함수로 전달하는 값을 '인자', 'argument'라고 하면 된다. 그리고 이 값을 받아서 함수 안으로 매개해주는 변수들을 매개변수, 영어로는 parameter라고 부른다. 이 두 가지는 함수로 들어오는 입력값이라고 보면 된다.


25. 함수의 기본

함수는 입력과 출력, 두 가지로 이루어져 있는 것이라고 말했다. 이번에는 출력을 배울 것이다. 그전에 '표현식(expression)'이라는 것이 무엇인가.

  • > 1+1
  • 2
1+1은 숫자 2에 대한 표현식이다. 2-1도 숫자 1에 대한 표현식이다.
  • > 2-1
  • 1
  • > 2-1
  • 1
  • 1===1
  • true
그러면 1과 1이 같으면? 이라는 것은, 좌항과 우항을 비교했을 때 같아야 한다. 같나? 같다. 그래서 true가 나온다. 그러면, 1===1은 true가 나오기 위한 표현식이다.

함수도 마찬가지이다.

  • sum(2, 3)
  • 5
sum(2, 3)을 5가 되는 표현식으로 만들고 싶다. 그렇게 하기 위해서는 return이라는 것을 알아야 한다. 


    <script>
    function sum(left, right){
        document.write(left+right+'<br>');
    }
    sum(2, 3);
    sum(3, 4);
    <script>
우리가 전에 만들어둔 sum(2, 3) 함수를 보자. 이 코드는 굉장히 편리하다. 어떤 점이 편리하냐 하면, 그냥 sum이라고 하면, 화면에다가 document.write(left+right+'<br>');까지 알아서 해준다. 기능이 상당히 많다. 첫번째 기능은, 좌항과 우항을 더하고, 두번째 기능은 br 태그를 뒤에 추가해서 줄바꿈을 한다. 그리고 마지막 기능은, 화면에다 써주는 역할까지 한다.

이게 딱 얘만 필요하다는 게 확실하면, 이걸로 됐다. 나쁜 코드가 아니다. 그러나, 우리가 무언가를 더 하는 작업이 매우 복잡하고 힘든 일이면서 그렇게 더해진 결과를 우리가 다양한 방법으로 사용해야 한다면, 우리는 이 코드를 예를 들어서 색깔을 바꿔야 한다. 그러면, 이렇게 하면 된다. 


        <script>
        function sumColorRed(left, right){
            document.write('<div style="color:red">+left+right+'</div><br>');
        }
        sum(2, 3);
        sumColorRed(2, 3);
        sum(3, 4);
        <script>
똑같은 결과인데, 우리가 출력하고 싶어서 이렇게 한다. sumColorRed(2, 3); 더하기가 다양한 곳에서 활용된다면, 이것은 상당히 많은 함수를 만들어야 한다는 뜻이 된다. 바로 이런 때에 return이 우리를 구원해줄 수 있다. 

            <script>
            function sum2(left, right){

            }
            sum2(2, 3);
            <script>
이때, sum2(2, 3);가 5에 대한 표현식이 되도록 하고 싶다. 그리고 다음과 같이 결과를 출력하고 싶다. 

                <script>
                function sum2(left, right){
    
                }
                document.write(sum2(2, 3)+'<br>');
                document.write('<div style="color:red">+sum2(2, 3)+'</div>');
                document.write('<div style="font-size:3rem">+sum2(2, 3)+'</div>');
                <script>
이런 식으로, sum2을 실행한 결과가 무언가를 실행하는 것이 아니라, 이 sum2한 값을 돌려준다는 것이다. (돌려준다는 return이다.) 어떤 내용을? left+right의 값을. 그래서 이렇게 써준다. 

                    <script>
                    function sum2(left, right){
                        return left+right;
                    }
                    document.write(sum2(2, 3)+'<br>');
                    document.write('<div style="color:red">+sum2(2, 3)+'</div>');
                    document.write('<div style="font-size:3rem">+sum2(2, 3)+'</div>');
                    <script>
우리는 계산이라고 하는 기능만을, sum2라는 함수가 구현함으로써 저 원자화된 기능을 다양한 맥락 안에서 쓸 수 있는 가능성을 제공한다. 이것이 return이 가져다주는 엄청난 효용이다. 보는 것처럼, 'function'은 함수라는 것을 우리에게 알려주고 있고, (left, right)라고 매개변수로 들어있는 값을 return을 통해 출력함으로써 다양한 용도로 함수를 활용할 수 있게 된다라는, 너무 중요한 얘기를 해주고 있다.


28. 함수의 활용

지금부터 night-day버튼을 함수를 활용해서 코드를 효율적으로 바꾸는 작업을 해보겠다. 동작하는 내용은 똑같지만 코드를 똑같이 하는 것을 리팩토링이라고 한다. 그리고 리팩토링의 굉장히 중요한 대상 중에 하나, 수단중의 하나를 함수라고 한다. 함수는 리팩토링에서 굉장히 중요한 역할을 차지한다.

input 버튼(안에 js 코드가 길게 들어있는)이 1억개가 있다면 나쁜 일이 많이 생긴다. 그래서 이 js 코드를 중복되는 코드를 카피해서, head 태그 안쪽에 붙여넣기 했다. 그리고 웹브라우저에게, 이 일련의 코드들이 nightDayHandler라는 이름의 함수다, 라고 웹브라우저에게 알려줄 것이다. 


        <!doctype html>
        <html>
        <head>
        <meta charset="utf-8">
            <title></title>
            <script>
               function nightDayHandler(){
              var target = document.querySelector('body') 
              if(this.value === 'night'){
                              target.style.backgroundColor = 'black';
                              target.style.color = 'white';
                              this.value = 'day';
      
                          var alist = document.querySelectorAll('a');
                          var i = 0;
                          while(i < alist.length){
                              alist[i].style.color = 'powderblue';
                              i = i + 1;
                          }
                      } else {
                          target.style.backgroundColor = 'white';
                          target.style.color = 'black';
                          this.value = 'night';
      
                          var alist = document.querySelectorAll('a');
                          var i = 0;
                          while(i < alist.length){
                              alist[i].style.color = 'blue';
                              i = i + 1;
                          }
                        }
                }
            </script>
        </head>
        <body>
          <h1>web</h1>
              <input type="button" value="night" onclick="
                nightDayHandler();
              ">
이 일련의 로직들은 함수이다. 그러나 클릭해도 변화가 안 생기고, 다시 클릭하니 변화가 생긴다. 그리고 night와 day 문자가 안바뀐다. 왜 그러냐. 여기 있는 onclick 안에서의 this라는 것은 이 이벤트가 소속되어있는 태그를 가리키도록 약속되어있는데, 우리가 독립된 함수를 만들게 되면 저기있는 this는 이제 input 버튼이 아니고, 전역 객체를 가리키게 된다. 웹브라우저에서는 윈도우가 된다. (이해 못한다. 지금은. ) 그래서 this가 input을 가리키게 하기 위해서 nightDayHandler를 실행할 때, 여기에서 'this' 값을 준다. 

                            </head>
                            <body>
                              <h1>web</h1>
                                  <input type="button" value="night" onclick="
                                    nightDayHandler(this);
                                  ">
그러면 이 this값은 nightDayHandler가 실행될 때, 여기로 들어오는 매개할 때 쓰는 변수인 매개변수, 이 이름을 self라고 정의하겠다. 저 self로 this를 바꿀 것이다. 그리고 실행해보면 잘 동작하는 것을 볼 수 있다.
